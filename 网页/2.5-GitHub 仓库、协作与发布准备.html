2.5 GitHub 仓库、协作与发布准备

这一课把 GitHub 看成项目的公共存档、协作入口和部署触发点。

已整理 核心课

本节目标

把仓库托管、协作和部署准备连成一条线。

核心要点

  • 仓库不是代码抽屉，而是项目事实来源。
  • 把页面目录、课程目录和工作流放进仓库，才能让部署变成可重复过程。
  • 对个人项目来说，GitHub 也是最自然的发布备份中心。

本节练习

说明为什么“发布配置也要放进仓库”比口头说明更可靠。
