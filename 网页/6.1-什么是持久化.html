🎯 本节目标

理解数据持久化的概念、重要性和实现方式

📖 什么是持久化？

一句话解释

持久化 = 让数据"活下来"，即使关机、刷新也不会丢失

生活化比喻

想象你在写日记：

没有持久化 有持久化
写在沙滩上，浪一来就没了 写在日记本里，永久保存
写在白板上，擦掉就没了 拍照存手机，随时能看
记在脑子里，睡一觉可能忘 记在笔记本，想看就看

持久化就是把"临时记忆"变成"永久记忆"！

❓ 为什么需要持久化？

场景1：用户登录

没有持久化：用户登录 → 刷新页面 → 又要重新登录 😫 有持久化：用户登录 → 刷新页面 → 还是登录状态 ✅

场景2：购物车

没有持久化：加入购物车 → 关闭浏览器 → 购物车清空 😫 有持久化：加入购物车 → 关闭浏览器 → 购物车还在 ✅

场景3：用户设置

没有持久化：设置深色模式 → 刷新页面 → 又变回浅色 😫 有持久化：设置深色模式 → 刷新页面 → 还是深色 ✅

总结

  1. 保存用户数据 - 账号、设置、历史记录
  2. 提升用户体验 - 不用重复操作
  3. 数据不丢失 - 关机、断网也没关系
  4. 跨设备同步 - 手机电脑都能用（需要服务器）

🎯 哪些场景需要持久化？

必须持久化的场景

场景 原因 推荐方式
用户登录状态 不能每次都重新登录 Cookies + 服务器
用户账号数据 用户信息不能丢 服务器数据库
付款/订单信息 涉及金钱，绝对不能丢 服务器数据库
用户上传的文件 用户的重要资料 云存储

强烈建议持久化的场景

场景 用户体验影响 推荐方式
编辑一半的内容 用户辛苦写的东西丢了会很崩溃 localStorage 自动保存
表单填写一半 填了很多信息突然没了 sessionStorage
购物车 选了半天的商品没了 localStorage 或服务器
用户偏好设置 每次都要重新设置很烦 localStorage

📦 客户端存储方式详解

1. localStorage（本地存储）

比喻：像你家里的保险箱，东西放进去，关门也不会丢

特点说明
容量约 5MB
有效期永久（除非手动删除）
适合存用户设置、主题偏好、草稿
// 保存数据 localStorage.setItem('theme', 'dark'); // 读取数据 const theme = localStorage.getItem('theme'); // 删除数据 localStorage.removeItem('theme');

2. sessionStorage（会话存储）

比喻：像酒店的保险箱，退房（关闭标签页）就清空

特点说明
容量约 5MB
有效期关闭标签页就清空
适合存临时表单数据、一次性信息

3. Cookies（小饼干）

比喻：像会员卡，每次去店里都要出示

特点说明
容量约 4KB（很小！）
有效期可设置过期时间
适合存登录状态、用户标识

4. IndexedDB（索引数据库）

比喻：像一个小型数据库，可以存很多复杂数据

特点说明
容量很大（几百MB甚至更多）
有效期永久
适合存离线应用数据、大量结构化数据

📝 常见持久化场景代码示例

场景1：保存主题设置

这就是我们课程网页用的方法！

// 保存主题设置 function saveTheme(theme) { localStorage.setItem('theme', theme); } // 读取主题设置 function loadTheme() { return localStorage.getItem('theme') || 'light'; } // 页面加载时应用主题 const savedTheme = loadTheme(); if (savedTheme === 'dark') { document.body.classList.add('dark'); } // 切换主题时保存 function toggleTheme() { document.body.classList.toggle('dark'); const newTheme = document.body.classList.contains('dark') ? 'dark' : 'light'; saveTheme(newTheme); }

场景2：编辑内容自动保存

问题：用户写了很长的文章，不小心关闭页面，内容全没了！

// 获取编辑器内容 const editor = document.getElementById('editor'); // 每 3 秒自动保存 setInterval(() => { localStorage.setItem('draft', editor.value); console.log('草稿已自动保存'); }, 3000); // 页面加载时恢复草稿 window.onload = function() { const draft = localStorage.getItem('draft'); if (draft) { editor.value = draft; console.log('已恢复草稿'); } };

场景3：页面滚动位置记忆

问题：用户看到一半，点了其他链接，返回后又要从头找

// 离开页面前保存滚动位置 window.onbeforeunload = function() { sessionStorage.setItem('scrollPosition', window.scrollY); }; // 返回页面时恢复位置 window.onload = function() { const position = sessionStorage.getItem('scrollPosition'); if (position) { window.scrollTo(0, parseInt(position)); } };

💬 如何告诉 AI 需要持久化？

描述需求的模板

模板1：基础描述 我需要保存 [什么数据]， 当用户 [什么操作] 后， 即使 [刷新页面/关闭浏览器/换设备]， 数据也要保留。 模板2：具体场景 场景：用户在编辑框里写内容 需求： 1. 每隔 5 秒自动保存草稿 2. 刷新页面后能恢复之前写的内容 3. 用户点击"发布"后清除草稿 请用 localStorage 实现

实际例子

例子1：主题设置 我的网页有深色/浅色模式切换， 我希望用户选择后，刷新页面还是保持用户选择的模式， 请用 localStorage 保存用户的主题偏好。 例子2：购物车 我在做一个购物网站， 用户添加商品到购物车后， 关闭浏览器再打开，购物车里的商品还要在， 请帮我实现购物车的持久化存储。

关键词提示

你想要的效果可以说的关键词
刷新不丢失持久化、保存状态、记住设置
自动保存自动保存、草稿功能、实时保存
恢复数据恢复、还原、读取保存的数据
跨页面保持全局状态、跨页面共享
关闭后保留永久保存、长期存储

⚠️ 持久化注意事项

1. 不要存敏感信息

❌ 不要存：密码、银行卡号、身份证号 ✅ 可以存：用户偏好设置、草稿内容、非敏感数据

2. 提供清除数据的选项

让用户可以主动清除保存的数据：

function clearAllSavedData() { localStorage.clear(); sessionStorage.clear(); alert('所有保存的数据已清除'); }
