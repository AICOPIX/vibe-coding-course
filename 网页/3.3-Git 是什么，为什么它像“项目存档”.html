3.3 Git 是什么，为什么它像“项目存档”

Git 的价值不在于显得专业，而在于让项目历史可追溯、可比较、可回退。

已整理 核心课

本节目标

理解版本管理的最小闭环。

核心要点

  • 提交的本质是保存一个清晰的阶段状态。
  • 对比改动能帮助你判断 AI 到底改了什么。
  • 小而明确的提交，比大而混乱的提交更容易维护。

本节练习

写出一条你认同的提交信息规范，并解释为什么这样更容易维护。
