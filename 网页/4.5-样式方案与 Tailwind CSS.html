4.5 样式方案与 Tailwind CSS

样式方案的关键不是跟风，而是保持一致、可维护和可快速调整。

已整理 核心课

本节目标

理解常见样式组织方式，并知道 Tailwind 的价值与边界。

核心要点

  • 原生 CSS 适合简单站点和结构明确的项目。
  • Tailwind 适合快速搭建统一视觉，但也需要清晰的样式约束。
  • 无论使用哪种方案，命名、层次和响应式规则都必须稳定。

本节练习

比较“共享 CSS 文件”与“全部写内联样式”的维护差异。

内容说明

基于旧 4.11。
