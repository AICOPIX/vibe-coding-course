2.4 Git 基础与提交协作

Git 的作用不是增加复杂度，而是为项目提供可追溯、可比较、可回退的历史。

已整理 核心课

本节目标

理解版本管理的最小闭环。

核心要点

  • 提交的本质是保存一个清晰的阶段状态。
  • 对比改动可以帮助你判断 AI 改了什么，而不是盲目接受。
  • 一个小而明确的提交，比一个大而混乱的提交更容易维护。

本节练习

写出你认同的一条提交信息规范，例如“只做一件事，并在信息中说明结果”。
