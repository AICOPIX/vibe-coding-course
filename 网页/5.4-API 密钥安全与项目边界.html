5.4 API 密钥安全与项目边界

安全的起点不是会多少加密，而是先知道哪些数据绝不能裸露。

已整理 核心课

本节目标

明确什么可以放前端，什么必须避开前端。

核心要点

  • API 密钥不应直接写进公开网页。
  • 静态站点适合展示和公开内容，不适合保存敏感逻辑。
  • 项目边界越清楚，后续技术选择越不容易失控。

本节练习

列出 3 类绝对不该出现在公开 HTML 文件里的信息。
