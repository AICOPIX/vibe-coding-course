## 💻 第一个功能：编写代码

### 场景 1：简单的代码

假设你想创建一个 JavaScript 函数，计算两个数的和。

**你可以说**：

```
创建一个 JavaScript 函数，计算两个数的和。
函数名叫 add，接收两个参数 a 和 b。
```

**Codex 会生成**：

```javascript
function add(a, b) {
  return a + b;
}
```

### 场景 2：复杂的代码

假设你想创建一个 React 组件，显示用户列表。

**你可以说**：

```
创建一个 React 组件，显示用户列表。
需要：
- 从 API 获取用户数据
- 显示用户名、邮箱、注册日期
- 使用 Tailwind CSS 美化
- 添加加载状态
- 添加错误处理
```

**Codex 会生成**：

```javascript
import React, { useState, useEffect } from 'react';

export default function UserList() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetchUsers();
  }, []);

  const fetchUsers = async () => {
    try {
      const response = await fetch('/api/users');
      const data = await response.json();
      setUsers(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  if (loading) return <div>加载中...</div>;
  if (error) return <div>错误：{error}</div>;

  return (
    <div className="p-4">
      <h1 className="text-2xl font-bold mb-4">用户列表</h1>
      <table className="w-full border">
        <thead>
          <tr className="bg-gray-100">
            <th className="p-2">用户名</th>
            <th className="p-2">邮箱</th>
            <th className="p-2">注册日期</th>
          </tr>
        </thead>
        <tbody>
          {users.map(user => (
            <tr key={user.id} className="border-t">
              <td className="p-2">{user.name}</td>
              <td className="p-2">{user.email}</td>
              <td className="p-2">{user.createdAt}</td>
            </tr>
          ))}
        </tbody>
      </table>
    </div>
  );
}
```

### 场景 3：修改现有代码

假设你有一个函数，想让 Codex 改进它。

**你可以说**：

```
这是我的函数：

function calculateTotal(items) {
  let total = 0;
  for (let i = 0; i < items.length; i++) {
    total += items[i].price;
  }
  return total;
}

请用现代 JavaScript 改进这个函数。
```

**Codex 会改进为**：

```javascript
function calculateTotal(items) {
  return items.reduce((total, item) => total + item.price, 0);
}
```

## 🐛 第二个功能：修复 Bug

### 场景 1：找出问题

假设你的代码有 Bug，但你不知道问题在哪里。

**你可以说**：

```
这个函数有问题。当我输入空数组时，它返回 undefined。

function getMax(arr) {
  return Math.max(...arr);
}

请找出问题并修复。
```

**Codex 会分析并修复**：

```javascript
function getMax(arr) {
  if (arr.length === 0) return null;
  return Math.max(...arr);
}
```

**解释**：
- 问题：空数组会导致 Math.max 返回 -Infinity
- 解决：添加检查，如果数组为空返回 null

### 场景 2：性能问题

假设你的代码很慢。

**你可以说**：

```
这个函数很慢。我有 10,000 个用户，
查询需要 5 秒。怎么优化？

function findUser(users, id) {
  for (let i = 0; i < users.length; i++) {
    if (users[i].id === id) {
      return users[i];
    }
  }
  return null;
}
```

**Codex 会建议**：

```javascript
// 方案 1：使用 Map（最快）
const userMap = new Map(users.map(u => [u.id, u]));
function findUser(id) {
  return userMap.get(id) || null;
}

// 方案 2：使用 find（简单）
function findUser(users, id) {
  return users.find(u => u.id === id) || null;
}
```

### 场景 3：安全问题

假设你的代码有安全漏洞。

**你可以说**：

```
请检查这个代码有没有安全问题：

app.post('/login', (req, res) => {
  const user = db.query(
    'SELECT * FROM users WHERE email = ' + req.body.email
  );
  if (user.password === req.body.password) {
    res.send('登录成功');
  }
});
```

**Codex 会指出问题**：

```
我发现了两个严重的安全问题：

1. SQL 注入漏洞
   问题：直接拼接用户输入到 SQL 查询
   解决：使用参数化查询

2. 密码明文存储
   问题：密码没有加密
   解决：使用 bcrypt 加密

修复后的代码：

app.post('/login', (req, res) => {
  const user = db.query(
    'SELECT * FROM users WHERE email = ?',
    [req.body.email]
  );
  if (bcrypt.compareSync(req.body.password, user.password)) {
    res.send('登录成功');
  }
});
```

## 📖 第三个功能：理解代码

### 场景 1：解释函数

假设你看到一个复杂的函数，不知道它是干什么的。

**你可以说**：

```
请解释这个函数是干什么的：

function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}
```

**Codex 会解释**：

```
这个函数计算斐波那契数列的第 n 个数。

工作原理：
1. 如果 n 小于等于 1，直接返回 n
2. 否则，返回前两个数的和

例子：
- fibonacci(0) = 0
- fibonacci(1) = 1
- fibonacci(2) = 1
- fibonacci(3) = 2
- fibonacci(4) = 3
- fibonacci(5) = 5

注意：这个实现很慢，因为有很多重复计算。
更好的方法是使用动态规划或迭代。
```

### 场景 2：理解项目结构

假设你接手一个新项目，不知道从哪里开始。

**你可以说**：

```
请分析这个项目的结构，告诉我各个文件的作用：

src/
├── components/
│   ├── Header.jsx
│   ├── Footer.jsx
│   └── Card.jsx
├── pages/
│   ├── Home.jsx
│   └── About.jsx
├── utils/
│   ├── api.js
│   └── helpers.js
└── styles/
    └── index.css
```

**Codex 会分析**：

```
这是一个 React 应用，结构如下：

components/ - React 组件
  - Header.jsx：页面顶部导航
  - Footer.jsx：页面底部
  - Card.jsx：可复用的卡片组件

pages/ - 页面文件
  - Home.jsx：首页
  - About.jsx：关于页面

utils/ - 工具函数
  - api.js：API 调用函数
  - helpers.js：辅助函数

styles/ - 样式文件
  - index.css：全局样式

这是一个典型的 React 应用结构。
```

### 场景 3：学习新概念

假设你想学习某个编程概念。

**你可以说**：

```
请用简单的例子解释什么是闭包（Closure）。
```

**Codex 会解释**：

```
闭包是一个函数，它可以访问外层函数的变量。

简单例子：

function outer() {
  const message = '你好';
  
  function inner() {
    console.log(message);  // 可以访问 outer 的变量
  }
  
  return inner;
}

const greet = outer();
greet();  // 输出：你好

为什么有用？
1. 数据隐私 - 变量不会被外部访问
2. 工厂函数 - 创建多个相似的函数
3. 回调函数 - 保存状态

实际例子：

function createCounter() {
  let count = 0;
  
  return {
    increment: () => ++count,
    decrement: () => --count,
    getCount: () => count
  };
}

const counter = createCounter();
counter.increment();  // 1
counter.increment();  // 2
counter.decrement();  // 1
```

## 🎬 实际操作步骤

### 步骤 1：打开 Codex

选择你喜欢的方式：
- ChatGPT 网页版
- CLI 命令行
- IDE 扩展

### 步骤 2：输入清晰的需求

**好的需求**：
```
创建一个 Python 函数，检查一个数是否是质数。
函数应该：
- 接收一个整数参数
- 返回 True 或 False
- 处理边界情况（小于 2 的数）
```

**不好的需求**：
```
创建一个函数
```

### 步骤 3：等待 Codex 完成

Codex 会分析你的需求，生成代码。

### 步骤 4：查看和修改结果

- 如果满意，复制代码
- 如果不满意，给出反馈

**反馈例子**：
```
不对，我想要的是：
- 函数应该返回质数的个数，而不是 True/False
- 应该处理负数
```

## 💡 提示和技巧

### 技巧 1：给出上下文

告诉 Codex 你的项目背景。

**例子**：
```
我在做一个 React 应用，使用 TypeScript 和 Tailwind CSS。
请创建一个用户列表组件。
```

### 技巧 2：提供示例

给出输入和输出的例子。

**例子**：
```
创建一个函数，格式化日期。

输入：2025-01-17
输出：2025年1月17日

输入：2025-01-17T14:30:00
输出：2025年1月17日 下午2:30
```

### 技巧 3：分步骤进行

如果需求很复杂，分成几个小步骤。

**例子**：
```
第一步：创建一个 API 端点，获取用户列表

第二步：创建一个 React 组件，显示用户列表

第三步：添加搜索功能
```

### 技巧 4：要求解释

让 Codex 解释生成的代码。

**例子**：
```
创建一个函数，并解释它是怎么工作的。
```
