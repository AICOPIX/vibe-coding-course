## 📚 案例 1：创建个人博客网站

### 项目需求

创建一个个人博客网站，包含：
- 首页（展示最新文章）
- 文章列表页
- 文章详情页
- 关于我页面
- 搜索功能
- 评论功能

### 技术栈

- 前端：React + Tailwind CSS
- 后端：Node.js + Express
- 数据库：MongoDB
- 部署：Vercel + Heroku

## 💡 最佳实践

### 实践 1：分解大任务

不要一次性要求 Codex 完成整个项目。

**不好**：
```
创建一个完整的电商应用
```

**好**：
```
第一步：创建项目结构
第二步：创建数据库模型
第三步：创建 API 端点
...
```

### 实践 2：提供清晰的需求

给出尽可能多的细节。

**不好**：
```
创建一个登录页
```

**好**：
```
创建一个登录页，包含：
- 用户名输入框
- 密码输入框
- 记住我复选框
- 登录按钮
- 忘记密码链接
- 注册链接
使用 React 和 Tailwind CSS
```

### 实践 3：审查生成的代码

不要盲目接受 Codex 生成的代码。

- 理解代码逻辑
- 检查是否符合需求
- 测试功能
- 优化性能

### 实践 4：逐步迭代

如果不满意，给出反馈并要求修改。

**例子**：
```
这个设计不对。
应该是...

请修改。
```

### 实践 5：学习和理解

不要只是复制代码。

- 理解代码如何工作
- 学习新的编程概念
- 改进你的编程技能
