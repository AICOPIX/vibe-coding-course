## 🔍 第一个高级功能：代码审查

### 什么是代码审查？

代码审查是检查代码质量、找出 Bug 和安全问题的过程。

Codex 可以自动做这件事，比人工审查更快、更全面。

### 场景 1：审查单个函数

**你可以说**：

```
请审查这个函数，找出问题：

function getUserData(userId) {
  const user = db.query('SELECT * FROM users WHERE id = ' + userId);
  return user;
}
```

**Codex 会审查**：

```
我发现了以下问题：

1. 🔴 严重：SQL 注入漏洞
   问题：直接拼接用户输入到 SQL 查询
   风险：攻击者可以执行任意 SQL 命令
   解决：使用参数化查询
   
   修复：
   const user = db.query('SELECT * FROM users WHERE id = ?', [userId]);

2. 🟡 中等：缺少错误处理
   问题：如果查询失败，函数会崩溃
   解决：添加 try-catch
   
   修复：
   try {
     const user = db.query('SELECT * FROM users WHERE id = ?', [userId]);
     return user;
   } catch (error) {
     console.error('查询失败:', error);
     return null;
   }

3. 🟡 中等：缺少输入验证
   问题：没有检查 userId 是否有效
   解决：添加验证
   
   修复：
   if (!userId || typeof userId !== 'number') {
     throw new Error('无效的用户 ID');
   }

修复后的完整代码：
[完整代码...]
```

## 🔄 第二个高级功能：多步骤任务

### 什么是多步骤任务？

多步骤任务是指需要多个步骤才能完成的复杂任务。

Codex 可以自动完成所有步骤，而不需要你每次都给指令。

### 场景 1：创建完整的应用

**你可以说**：

```
我想创建一个完整的 Todo 应用。

需要：
1. 后端 API（Node.js + Express）
   - GET /todos - 获取所有 Todo
   - POST /todos - 创建新 Todo
   - PUT /todos/:id - 更新 Todo
   - DELETE /todos/:id - 删除 Todo

2. 前端（React）
   - 显示 Todo 列表
   - 添加新 Todo
   - 编辑 Todo
   - 删除 Todo
   - 标记为完成

3. 数据库（SQLite）
   - 创建 todos 表
   - 字段：id, title, description, completed, createdAt

4. 部署
   - 部署到 Heroku
   - 配置环境变量
```

**Codex 会自动完成**：

```
我会为你创建完整的应用。这需要以下步骤：

- 创建 SQLite 数据库

- 实现 DELETE /todos/:id

第四步：创建前端项目
- 初始化 React 项目

- 连接到后端 API

第五步：测试
- 测试所有 API 端点
- 测试前端功能

第六步：部署
- 部署到 Heroku
- 配置环境变量

开始执行...

## 💬 第三个高级功能：Slack 集成

### 什么是 Slack 集成？

Slack 集成让你可以在 Slack 中直接使用 Codex，而不需要打开网页或终端。

### 设置 Slack 集成

#### 步骤 1：访问 Codex Slack 集成页面

https://openai.com/codex/slack

#### 步骤 2：点击"添加到 Slack"

#### 步骤 3：选择你的 Slack 工作区

#### 步骤 4：授权

点击"授权"按钮，允许 Codex 访问你的 Slack。

#### 步骤 5：完成

Codex 会被添加到你的 Slack 工作区。

### 使用 Slack 集成

#### 在频道中使用

在任何 Slack 频道中，输入：

```
@Codex 创建一个登录页面
```

Codex 会：
1. 在 Slack 中收集信息
2. 在云端完成任务
3. 返回结果链接

#### 在私聊中使用

直接给 Codex 发私聊：

```
创建一个用户列表组件
```

#### 查看结果

Codex 会返回一个链接，你可以：
- 查看生成的代码
- 修改代码
- 下载代码
- 合并到项目

### Slack 集成的优点

1. **快速** - 不需要切换应用
2. **协作** - 整个团队可以看到
3. **历史记录** - 所有对话都保存在 Slack
4. **集成** - 与其他 Slack 工具集成

## ⚡ 第四个高级功能：性能优化

### 什么是性能优化？

性能优化是指让应用运行更快、使用更少资源。

Codex 可以帮你分析和优化性能。

### 场景 1：优化数据库查询

**你可以说**：

```
这个查询很慢。怎么优化？

SELECT * FROM users 
WHERE created_at > '2024-01-01' 
AND status = 'active'
ORDER BY created_at DESC
LIMIT 100;
```

**Codex 会建议**：

```
我发现了几个性能问题：

1. 缺少索引
   问题：没有为 created_at 和 status 建立索引
   解决：添加索引
   
   CREATE INDEX idx_users_created_status 
   ON users(created_at, status);

2. 查询所有列
   问题：SELECT * 会获取不需要的列
   解决：只查询需要的列
   
   SELECT id, name, email, status 
   FROM users 
   WHERE created_at > '2024-01-01' 
   AND status = 'active'
   ORDER BY created_at DESC
   LIMIT 100;

3. 排序可能很慢
   问题：如果没有索引，排序会很慢
   解决：确保有索引

优化后的查询：
[优化后的 SQL...]
```

### 场景 2：优化 React 组件

**你可以说**：

```
这个 React 组件很慢。怎么优化？

[粘贴组件代码]
```

**Codex 会建议**：

```
我发现了以下性能问题：

1. 没有使用 useMemo
   问题：每次渲染都重新计算
   解决：使用 useMemo 缓存结果

2. 没有使用 useCallback
   问题：子组件不必要的重新渲染
   解决：使用 useCallback 缓存函数

3. 没有使用 React.memo
   问题：即使 props 没变，也会重新渲染
   解决：使用 React.memo 包装组件

4. 列表没有 key
   问题：React 无法追踪列表项
   解决：为每个列表项添加唯一的 key
