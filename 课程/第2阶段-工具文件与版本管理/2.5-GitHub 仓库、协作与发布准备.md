# 2.5 GitHub 仓库、协作与发布准备

> 学习状态：已整理
> 所属阶段：第2阶段 工具、文件与版本管理
> 内容说明：合并旧 2.10、2.11、2.12。

## 本节目标

把仓库托管、协作和部署准备连成一条线。

## 课程摘要

这一课把 GitHub 看成项目的公共存档、协作入口和部署触发点。

## 核心要点

- 仓库不是代码抽屉，而是项目事实来源。
- 把页面目录、课程目录和工作流放进仓库，才能让部署变成可重复过程。
- 对个人项目来说，GitHub 也是最自然的发布备份中心。

## 本节练习

说明为什么“发布配置也要放进仓库”比口头说明更可靠。

## 学完后你应该能做到

- 用自己的话解释本节核心概念
- 把本节内容放回当前项目里判断它的实际作用
- 在后续页面或项目中继续复用这里的方法
