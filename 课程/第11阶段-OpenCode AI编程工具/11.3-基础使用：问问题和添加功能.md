## 💬 第一个功能：问问题

### 场景 1：理解代码

你有一个项目，但不太理解某个文件或函数的作用。

**你可以问**：

```
> 请解释一下 src/utils/auth.js 这个文件是干什么的？
```

**OpenCode 会**：
1. 读取这个文件
2. 分析代码
3. 用中文给你解释

**例子回答**：
```
这个文件是身份验证工具库。它包含三个主要函数：

1. validateToken() - 验证用户的登录令牌是否有效
2. generateToken() - 为新用户生成登录令牌
3. refreshToken() - 刷新过期的令牌

这些函数用于保护你的应用，确保只有登录的用户才能访问某些功能。
```

### 场景 2：理解整个项目

```
> 这个项目的整体结构是什么？各个文件夹的作用是什么？
```

**OpenCode 会**：
1. 扫描整个项目
2. 分析文件结构
3. 给出项目概览

### 场景 3：找出问题

```
> 为什么这个函数会报错？
> 这里的代码有什么问题吗？
> 这个逻辑对吗？
```

**OpenCode 会**：
1. 分析代码
2. 找出潜在问题
3. 给出建议

## ✨ 第二个功能：添加新功能

### 场景 1：简单的功能添加

假设你有一个网站，想添加一个"关于我们"页面。

**你可以说**：

```
> 给我的网站添加一个"关于我们"页面。
> 页面应该包含：
> - 公司简介
> - 团队成员介绍
> - 联系方式
> 使用 HTML 和 CSS，要求响应式设计
```

**OpenCode 会**：
1. 创建新的 HTML 文件
2. 写入页面代码
3. 添加 CSS 样式
4. 确保在手机和电脑上都能正常显示

### 场景 2：复杂的功能添加

```
> 给我的应用添加用户登录功能。
> 需要：
> - 登录表单（用户名和密码）
> - 密码验证
> - 登录成功后跳转到首页
> - 登录失败显示错误信息
> 使用 React 和 Node.js
```

**OpenCode 会**：
1. 创建登录表单组件
2. 写后端验证逻辑
3. 处理错误情况
4. 连接前后端

### 场景 3：修改现有功能

```
> 修改首页的导航栏。
> 需要：
> - 添加"产品"菜单项
> - 添加"博客"菜单项
> - 在右侧添加"登录"按钮
> - 添加悬停效果
```

**OpenCode 会**：
1. 找到导航栏代码
2. 添加新菜单项
3. 添加样式和交互效果

## 🎬 实际操作步骤

### 步骤 1：启动 OpenCode

```bash
opencode
```

### 步骤 2：输入你的需求

在 `>` 提示符后输入你的需求。

**好的需求示例**：
```
> 创建一个 React 组件，显示用户列表。
> 列表应该包含：
> - 用户名
> - 邮箱
> - 注册日期
> 使用 Tailwind CSS 美化样式
```

**不好的需求示例**：
```
> 创建一个组件
```
（太模糊，OpenCode 不知道你想要什么）

### 步骤 3：等待 OpenCode 完成

OpenCode 会：
1. 分析你的需求
2. 生成代码
3. 创建或修改文件
4. 显示完成信息

### 步骤 4：查看结果

打开生成的文件，检查代码是否符合你的需求。

## 🔄 修改和撤销

### 场景：OpenCode 做的不对

假设你要求 OpenCode 添加一个功能，但结果不满意。

**你可以**：

#### 方法 1：撤销（Undo）

```
> /undo
```

这会：
1. 撤销 OpenCode 的所有修改
2. 恢复到之前的状态
3. 显示你之前的消息

然后你可以修改需求，重新尝试：

```
> /redo
```

这会重新执行之前的操作。

#### 方法 2：给出反馈

```
> 不对，我想要的是...
> 请改成...
> 这里应该...
```

OpenCode 会根据你的反馈修改代码。

**例子**：

```
> 创建一个登录表单

[OpenCode 创建了表单]

> 不对，我想要的是：
> - 表单应该在页面中间
> - 背景应该是白色
> - 按钮应该是蓝色
> - 添加"忘记密码"链接

[OpenCode 修改表单]
```

## 📸 拖拽图片

OpenCode 还支持一个很酷的功能：**拖拽图片**。

### 使用场景

你有一个设计稿（图片），想让 OpenCode 根据设计稿生成代码。

### 操作步骤

1. 在 OpenCode 中输入你的需求
2. 将设计稿图片**拖拽**到终端窗口
3. OpenCode 会分析图片
4. 根据设计稿生成代码

**例子**：

```
> 根据这个设计稿创建网页

[拖拽设计稿图片到这里]

OpenCode 会：
1. 分析设计稿
2. 识别颜色、布局、字体
3. 生成对应的 HTML 和 CSS
```

## 🎮 两种工作模式

### 模式 1：Build（构建模式）

OpenCode 直接修改你的代码。

**何时使用**：
- 你已经确定要做什么
- 你信任 OpenCode 的方案

**切换方式**：按 `Tab` 键

**界面显示**：
```
Build Mode  ← 你在这里
```

### 模式 2：Plan（计划模式）

OpenCode 只给出计划，不修改代码。

**何时使用**：
- 你想先看看方案
- 你想给出反馈
- 你想确保方向正确

**切换方式**：按 `Tab` 键

**界面显示**：
```
Plan Mode  ← 你在这里
```

### 如何使用 Plan 模式

#### 步骤 1：切换到 Plan 模式

按 `Tab` 键，看到 `Plan Mode` 提示。

#### 步骤 2：描述你的需求

```
> 我想给网站添加一个搜索功能
```

#### 步骤 3：OpenCode 给出计划

```
我会这样实现搜索功能：

第一步：创建搜索输入框
- 在导航栏添加搜索框
- 使用 HTML input 元素

第二步：添加搜索逻辑
- 当用户输入时，实时搜索
- 显示匹配的结果

第三步：美化样式
- 添加 CSS 样式
- 添加悬停效果

你觉得这个方案怎么样？
```

#### 步骤 4：给出反馈

```
> 很好，但是搜索结果应该显示在下拉菜单里
> 而不是在页面下方
```

#### 步骤 5：OpenCode 调整计划

```
好的，我会修改方案：

第一步：创建搜索输入框
- 在导航栏添加搜索框

第二步：创建下拉菜单
- 搜索结果显示在下拉菜单中
- 用户点击结果可以跳转

...
```

#### 步骤 6：确认后切换到 Build 模式

```
> 好的，开始构建吧
```

按 `Tab` 键切换到 Build 模式，OpenCode 开始写代码。

## 💡 提示和技巧

### 技巧 1：给出足够的细节

**不好**：
```
> 创建一个表单
```

**好**：
```
> 创建一个用户注册表单，包含：
> - 用户名（必填）
> - 邮箱（必填，需要验证格式）
> - 密码（必填，至少8个字符）
> - 确认密码（必填，需要与密码一致）
> - 同意条款复选框
> - 提交按钮
> 使用 React 和 Tailwind CSS
```

### 技巧 2：像和同事交流一样

想象你在和一个初级开发者交流。给出清晰的指示，但不需要太技术性。

**例子**：
```
> 给这个按钮添加一个加载状态。
> 当用户点击时，按钮应该显示"加载中..."
> 并且禁用点击，直到请求完成。
```

### 技巧 3：分步骤进行

如果需求很复杂，分成几个小步骤。

**例子**：

```
> 第一步：创建一个用户列表页面
> 显示所有用户的名字和邮箱

[等待完成]

> 第二步：添加删除用户的功能
> 每个用户旁边有一个删除按钮

[等待完成]

> 第三步：添加编辑用户的功能
> 点击用户可以编辑信息
```

### 技巧 4：使用 `/share` 分享

如果你想和别人分享你的对话：

```
> /share
```

OpenCode 会生成一个链接，你可以分享给团队。

## 🔧 常用命令

| 命令 | 作用 |
|------|------|
| `/undo` | 撤销上一步修改 |
| `/redo` | 重做上一步修改 |
| `/share` | 分享当前对话 |
| `/clear` | 清空对话历史 |
| `Tab` | 切换 Build/Plan 模式 |
| `Ctrl+C` | 退出 OpenCode |
