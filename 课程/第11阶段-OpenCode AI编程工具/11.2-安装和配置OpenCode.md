# 11.2 安装和配置 OpenCode

## 📋 前置要求

在安装 OpenCode 之前，你需要准备：

### 1. 一个现代化的终端
如果你想用终端版本，需要一个好用的终端：

**Windows**：
- Windows Terminal（推荐）
- PowerShell 7+

**macOS**：
- Terminal（系统自带）
- iTerm2（更强大）

**Linux**：
- GNOME Terminal
- Konsole
- 其他 Linux 终端

### 2. 一个 AI 模型的 API 密钥
你需要选择一个 AI 模型提供商，并获取 API 密钥。

**推荐方案**：使用 **OpenCode Zen**（官方推荐）
- 这是一个经过测试的模型列表
- 不需要自己配置复杂的参数
- 最简单的方式

---

## 🔑 第一步：获取 API 密钥

### 方案 A：使用 OpenCode Zen（推荐）

**步骤**：

1. 打开 OpenCode 后，运行命令：`/connect`
2. 选择 `opencode`
3. 打开浏览器访问：https://opencode.ai/auth
4. 登录或注册账户
5. 添加支付信息（OpenCode Zen 是付费的，但很便宜）
6. 复制你的 API 密钥
7. 粘贴到 OpenCode 中

**优点**：
- 官方推荐
- 已经测试过最好的模型
- 不需要自己选择

### 方案 B：使用其他 AI 提供商

如果你已经有其他提供商的账户，也可以使用：

#### OpenAI（ChatGPT）
1. 访问 https://platform.openai.com/api/keys
2. 创建新的 API 密钥
3. 复制密钥

#### Anthropic（Claude）
1. 访问 https://console.anthropic.com/
2. 创建新的 API 密钥
3. 复制密钥

#### Google Gemini
1. 访问 https://aistudio.google.com/app/apikey
2. 创建新的 API 密钥
3. 复制密钥

---

## 💻 第二步：安装 OpenCode

### 方式 1：使用安装脚本（推荐）

这是最简单的方式。在你的终端运行：

```bash
curl -fsSL https://opencode.ai/install.sh | sh
```

**这个脚本会**：
- 自动检测你的操作系统
- 下载合适的版本
- 安装到正确的位置

### 方式 2：使用 Homebrew（macOS 和 Linux）

如果你已经安装了 Homebrew：

```bash
brew install opencode
```

### 方式 3：使用 AUR（Arch Linux）

如果你用 Arch Linux：

```bash
yay -S opencode
```

或者

```bash
pacman -S opencode
```

### 方式 4：使用 Go 安装

如果你已经安装了 Go：

```bash
go install github.com/sst/opencode@latest
```

### 方式 5：Windows 特殊说明

**注意**：Windows 上的自动安装还在开发中。

**推荐做法**：
1. 访问 https://github.com/sst/opencode/releases
2. 下载 `opencode-desktop-windows-x64.exe`
3. 运行安装程序

或者下载命令行版本的二进制文件。

### 方式 6：下载桌面应用

如果你不想用命令行，可以下载桌面版本：

1. 访问 https://opencode.ai/download
2. 选择你的操作系统
3. 下载并安装

**支持的系统**：
- macOS (Apple Silicon)
- macOS (Intel)
- Windows
- Linux (.deb, .rpm, AppImage)

## ⚙️ 第三步：配置 OpenCode

### 3.1 配置 API 密钥

安装完成后，你需要告诉 OpenCode 你的 API 密钥。

**方法 1：使用 `/connect` 命令（推荐）**

1. 打开 OpenCode
2. 输入命令：`/connect`
3. 选择你的 AI 提供商
4. 按照提示输入 API 密钥

**方法 2：使用环境变量**

你也可以设置环境变量。根据你选择的提供商：

```bash
# OpenAI
export OPENAI_API_KEY="your-api-key"

# Anthropic (Claude)
export ANTHROPIC_API_KEY="your-api-key"

# Google Gemini
export GEMINI_API_KEY="your-api-key"

# GitHub Copilot
export GITHUB_TOKEN="your-token"

# Groq
export GROQ_API_KEY="your-api-key"

# Azure OpenAI
export AZURE_OPENAI_ENDPOINT="your-endpoint"
export AZURE_OPENAI_API_KEY="your-api-key"
export AZURE_OPENAI_API_VERSION="2024-02-15-preview"
```

### 3.2 配置文件位置

OpenCode 会在以下位置查找配置文件（按优先级排列）：

| 位置 | 说明 | 优先级 |
|------|------|--------|
| 远程配置 | 组织提供的默认配置 | 最低 |
| `~/.config/opencode/opencode.json` | 全局配置（所有项目） | 中 |
| `OPENCODE_CONFIG` 环境变量 | 自定义配置路径 | 中高 |
| 项目根目录的 `opencode.json` | 项目特定配置 | 高 |
| `.opencode/` 目录 | 代理、命令、插件 | 高 |
| `OPENCODE_CONFIG_CONTENT` 环境变量 | 运行时覆盖 | 最高 |

**简单理解**：
- 全局配置 = 你所有项目的默认设置
- 项目配置 = 这个项目特有的设置
- 项目配置会覆盖全局配置

### 3.3 创建全局配置文件

创建文件：`~/.config/opencode/opencode.json`

**最简单的配置**：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": "anthropic",
  "model": "anthropic/claude-sonnet-4-5"
}
```

**更完整的配置**：

```json
{
  "$schema": "https://opencode.ai/config.json",
  
  // 选择 AI 提供商和模型
  "provider": "anthropic",
  "model": "anthropic/claude-sonnet-4-5",
  "small_model": "anthropic/claude-haiku-4-5",
  
  // 主题设置
  "theme": "opencode",
  
  // 自动更新
  "autoupdate": true,
  
  // 权限设置
  "permission": {
    "edit": "auto",
    "bash": "auto"
  }
}
```

### 3.4 创建项目配置文件

在你的项目根目录创建：`opencode.json`

```json
{
  "$schema": "https://opencode.ai/config.json",
  
  // 这个项目使用的模型
  "model": "anthropic/claude-sonnet-4-5",
  
  // 项目特定的权限
  "permission": {
    "bash": "ask"
  }
}
```

## 🧪 第四步：测试安装

### 验证安装成功

在终端运行：

```bash
opencode --version
```

你应该看到版本号，比如：`opencode version 0.1.0`

### 初始化项目

进入你的项目目录，运行：

```bash
cd /path/to/your/project
opencode init
```

这会：
1. 分析你的项目结构
2. 创建 `AGENTS.md` 文件
3. 帮助 OpenCode 理解你的项目

### 启动 OpenCode

```bash
opencode
```

你应该看到 OpenCode 的交互界面。

## 📍 配置文件优先级详解

### 例子 1：全局 + 项目配置

**全局配置** (`~/.config/opencode/opencode.json`)：
```json
{
  "theme": "opencode",
  "autoupdate": true,
  "model": "anthropic/claude-sonnet-4-5"
}
```

**项目配置** (`./opencode.json`)：
```json
{
  "model": "anthropic/claude-haiku-4-5"
}
```

**最终结果**：
```json
{
  "theme": "opencode",           // 来自全局
  "autoupdate": true,            // 来自全局
  "model": "anthropic/claude-haiku-4-5"  // 项目覆盖了全局
}
```

### 例子 2：使用环境变量覆盖

```bash
# 设置环境变量
export OPENCODE_CONFIG="/custom/path/config.json"

# 启动 OpenCode
opencode
```

这会使用自定义路径的配置文件。

## 🐛 常见问题

### Q1：我在 Windows 上安装失败了怎么办？

**A**：Windows 上的自动安装还在开发中。请：
1. 访问 https://github.com/sst/opencode/releases
2. 下载 Windows 版本的二进制文件
3. 手动运行

### Q2：我不知道选哪个 AI 模型

**A**：使用 **OpenCode Zen**（官方推荐）。它已经帮你选好了最好的模型。

### Q3：我的 API 密钥在哪里？

**A**：根据你选择的提供商：
- **OpenAI**：https://platform.openai.com/api/keys
- **Anthropic**：https://console.anthropic.com/
- **Google**：https://aistudio.google.com/app/apikey

### Q4：我可以同时使用多个 AI 模型吗？

**A**：可以！在配置文件中配置多个提供商，然后在 OpenCode 中切换。
