
## 📌 核心概念

**OpenCode** 是一个**开源 AI 编程代理**。简单来说，它就像一个住在你电脑里的"AI 编程助手"，可以帮你：

- 📝 理解代码
- ✨ 添加新功能
- 🐛 修复 Bug
- 💡 给出编程建议

## 🎯 OpenCode 能做什么

### 1. 问代码问题
你可以问 OpenCode 关于你项目的任何问题，它会帮你解释代码。

**例子**：
- "这个函数是干什么的？"
- "为什么这里会出错？"
- "这个项目的整体结构是什么？"

### 2. 添加新功能
告诉 OpenCode 你想要什么功能，它会帮你写代码。

**例子**：
- "给我的网站添加一个登录页面"
- "创建一个用户管理系统"
- "添加暗黑模式支持"

### 3. 制定计划（Plan 模式）
在真正写代码之前，OpenCode 可以先给你一个详细的计划。

**例子**：
- 你说："我想添加支付功能"
- OpenCode 回答："我会这样做：第一步...第二步...第三步..."
- 你确认后，它才开始写代码

### 4. 修改和撤销
如果 OpenCode 做的不对，你可以：
- 用 `/undo` 命令撤销
- 修改你的要求
- 让它重新尝试

## 🖥️ OpenCode 的三种使用方式

### 方式 1：终端（TUI）
在命令行里使用 OpenCode，像聊天一样交互。

**优点**：
- 快速
- 不需要打开其他软件
- 适合开发者

### 方式 2：桌面应用
下载 OpenCode 的桌面版本，像使用普通软件一样。

**优点**：
- 界面友好
- 不需要懂命令行
- 适合初学者

### 方式 3：IDE 扩展
在你的代码编辑器（如 VS Code）里直接使用 OpenCode。

**优点**：
- 不用切换窗口
- 集成度高
- 工作流顺畅

## 🤖 OpenCode 的两个工作模式

### 模式 1：Build（构建模式）
OpenCode 会直接修改你的代码。

**适合**：
- 你已经确定要做什么
- 你信任 OpenCode 的方案

### 模式 2：Plan（计划模式）
OpenCode 只给出计划，不修改代码。

**适合**：
- 你想先看看方案
- 你想给出反馈
- 你想确保方向正确

**切换方式**：按 `Tab` 键在两个模式之间切换

## 🔌 支持的 AI 模型

OpenCode 支持多个 AI 模型提供商：

| 提供商 | 模型示例 |
|--------|---------|
| **OpenAI** | GPT-4, GPT-4o |
| **Anthropic** | Claude 3.5 Sonnet, Claude Haiku |
| **Google** | Gemini 2.5 Pro |
| **GitHub Copilot** | GitHub Copilot 模型 |
| **AWS Bedrock** | Claude via AWS |
| **Groq** | 快速推理模型 |
| **Azure OpenAI** | Azure 托管的 OpenAI |

**推荐**：OpenCode 官方推荐使用 **OpenCode Zen**，这是一个经过测试和验证的模型列表。

## 📊 OpenCode vs Claude Code

| 特性 | OpenCode | Claude Code |
|------|----------|------------|
| **开源** | ✅ 是 | ❌ 否 |
| **本地运行** | ✅ 可以 | ❌ 不行 |
| **支持多个 AI 模型** | ✅ 是 | ❌ 仅 Claude |
| **终端使用** | ✅ 是 | ❌ 否 |
| **桌面应用** | ✅ 是 | ❌ 否 |
| **自定义配置** | ✅ 高度可定制 | ❌ 有限 |
