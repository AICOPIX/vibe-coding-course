## 📚 案例 1：创建一个个人作品集网站

### 项目需求

创建一个设计师的个人作品集网站，包含：
- 首页（介绍和作品展示）
- 关于页面
- 联系方式
- 响应式设计
- 暗黑模式支持

### 第一步：初始化项目

```bash
# 创建项目目录
mkdir my-portfolio
cd my-portfolio

# 初始化 OpenCode
opencode init
```

### 第二步：使用 Plan 模式制定计划

按 `Tab` 切换到 Plan 模式，输入：

```
> 我想创建一个设计师的个人作品集网站。
> 
> 需求：
> 1. 首页 - 展示我的最好的作品（3-5个）
> 2. 关于页面 - 介绍我的背景和技能
> 3. 联系页面 - 邮箱、社交媒体链接
> 4. 响应式设计 - 在手机、平板、电脑上都能正常显示
> 5. 暗黑模式 - 支持浅色和深色主题
> 
> 技术栈：HTML、CSS、JavaScript（不用框架）
> 
> 请给出你的实现计划
```

### 第三步：查看计划

OpenCode 会给出类似这样的计划：

```
我会这样实现你的作品集网站：

第一步：创建项目结构
- index.html（首页）
- about.html（关于页面）
- contact.html（联系页面）
- css/style.css（样式文件）
- js/script.js（交互脚本）
- images/（作品图片文件夹）

第二步：创建首页
- 导航栏（链接到其他页面）
- 英雄区（大标题和简介）
- 作品展示区（网格布局，3-5个作品）
- 页脚

第三步：创建其他页面
- 关于页面：个人介绍、技能列表
- 联系页面：邮箱、社交媒体链接

第四步：添加样式
- 使用 CSS Grid 和 Flexbox 实现响应式布局
- 添加暗黑模式支持

第五步：添加交互
- 暗黑模式切换按钮
- 平滑滚动
- 导航栏高亮

你觉得这个计划怎么样？
```

### 第四步：给出反馈

```
> 很好！但是我想要：
> 1. 首页的英雄区应该有一个背景图片
> 2. 作品应该有悬停效果
> 3. 导航栏应该是固定的（滚动时始终显示）
```

### 第五步：确认并开始构建

```
> 好的，开始构建吧
```

按 `Tab` 切换到 Build 模式，OpenCode 开始写代码。

### 第六步：查看结果

OpenCode 会创建所有文件。你可以：

```bash
# 用浏览器打开首页
open index.html
```

### 第七步：迭代和改进

如果有不满意的地方：

```
> 首页的作品卡片太小了，请增大尺寸
> 并且添加作品的描述文字
```

OpenCode 会修改代码。

## 🛒 案例 2：创建一个简单的待办事项应用

### 项目需求

创建一个 React 待办事项应用，包含：
- 添加任务
- 删除任务
- 标记完成
- 本地存储（刷新后数据不丢失）
- 任务统计

### 第一步：初始化 React 项目

```bash
# 创建 React 项目
npx create-react-app todo-app
cd todo-app

# 初始化 OpenCode
opencode init
```

### 第二步：使用 Plan 模式

```
> 我想创建一个待办事项应用。
> 
> 功能需求：
> 1. 输入框 - 用户可以输入新任务
> 2. 添加按钮 - 点击添加任务到列表
> 3. 任务列表 - 显示所有任务
> 4. 删除按钮 - 每个任务可以删除
> 5. 完成复选框 - 标记任务为完成
> 6. 任务统计 - 显示总任务数和完成数
> 7. 本地存储 - 刷新页面后数据保留
> 
> 使用 React Hooks（useState, useEffect）
> 使用 Tailwind CSS 美化
> 
> 请给出实现计划
```

### 第三步：查看计划并反馈

```
> 很好！但是我想要：
> 1. 添加一个"清空已完成任务"的按钮
> 2. 任务应该有优先级（高、中、低）
> 3. 不同优先级用不同颜色显示
```

### 第四步：开始构建

```
> 好的，开始构建
```

### 第五步：测试应用

```bash
npm start
```

在浏览器中测试应用。

### 第六步：迭代

如果有问题：

```
> 完成的任务应该显示删除线
> 并且文字颜色应该变淡
```

OpenCode 会修改样式。

## 🌐 案例 3：创建一个博客网站

### 项目需求

创建一个简单的博客网站，包含：
- 首页（文章列表）
- 文章详情页
- 分类功能
- 搜索功能
- 评论系统（简单版）

### 第一步：初始化项目

```bash
mkdir my-blog
cd my-blog
opencode init
```

### 第二步：制定计划

```
> 我想创建一个个人博客网站。
> 
> 功能需求：
> 1. 首页 - 显示最新的文章列表
> 2. 文章详情页 - 显示完整的文章内容
> 3. 分类 - 按分类浏览文章
> 4. 搜索 - 搜索文章标题和内容
> 5. 评论 - 简单的评论功能（存储在本地）
> 6. 响应式设计
> 7. 暗黑模式
> 
> 技术栈：React + Node.js + MongoDB（或简单的 JSON 文件存储）
> 
> 请给出实现计划
```

### 第三步：分步骤构建

由于这是一个复杂项目，分步骤进行：

#### 步骤 1：创建前端结构

```
> 第一步：创建 React 项目结构
> 
> 需要的组件：
> - Header（导航栏）
> - ArticleList（文章列表）
> - ArticleDetail（文章详情）
> - Sidebar（侧边栏，分类和搜索）
> - Comments（评论区）
> - Footer（页脚）
```

#### 步骤 2：创建后端 API

```
> 第二步：创建 Node.js 后端
> 
> 需要的 API 端点：
> - GET /api/articles - 获取所有文章
> - GET /api/articles/:id - 获取单篇文章
> - GET /api/articles?category=xxx - 按分类获取
> - GET /api/articles?search=xxx - 搜索文章
> - POST /api/comments - 添加评论
> - GET /api/comments/:articleId - 获取评论
```

#### 步骤 3：连接前后端

```
> 第三步：连接前后端
> 
> 在 React 中调用 API
> 显示真实的文章数据
> 实现搜索和分类功能
```

#### 步骤 4：添加样式和交互

```
> 第四步：美化界面
> 
> - 使用 Tailwind CSS
> - 添加暗黑模式
> - 添加加载动画
> - 添加错误提示
```

### 第四步：测试和部署

```bash
# 启动后端
npm run server

# 启动前端（新终端）
npm start
```

## 🎓 学习要点总结

### 1. 使用 Plan 模式的好处

- ✅ 在写代码前确认方向
- ✅ 避免做无用功
- ✅ 节省时间和 API 费用
- ✅ 可以提前发现问题

### 2. 分步骤进行复杂项目

- ✅ 先做简单的部分
- ✅ 逐步添加复杂功能
- ✅ 每一步都可以测试
- ✅ 更容易调试问题

### 3. 给出清晰的需求

**不好的需求**：
```
> 创建一个网站
```

**好的需求**：
```
> 创建一个个人作品集网站
> 包含首页、关于页面、联系方式
> 响应式设计，支持暗黑模式
> 使用 HTML、CSS、JavaScript
```

### 4. 充分利用反馈循环

```
需求 → 计划 → 反馈 → 修改计划 → 构建 → 测试 → 迭代
```

## 🚀 实战建议

### 建议 1：从小项目开始

不要一开始就做大项目。先做小项目，熟悉 OpenCode 的工作流程。

**推荐顺序**：
1. 静态网页（HTML + CSS）
2. 交互网页（HTML + CSS + JavaScript）
3. React 应用
4. 全栈应用（前端 + 后端）

### 建议 2：保存你的对话

使用 `/share` 命令保存对话链接，方便以后查看。

### 建议 3：版本控制

使用 Git 保存每个阶段的代码：

```bash
git add .
git commit -m "完成首页"

git add .
git commit -m "添加关于页面"

git add .
git commit -m "添加暗黑模式"
```

### 建议 4：定期测试

不要等到最后才测试。每完成一个功能就测试一次。

### 建议 5：记录问题

如果 OpenCode 做错了什么，记录下来：
- 什么需求
- 什么结果
- 怎么修复的

这样可以帮助你更好地描述需求。

## 📋 常见问题

### Q1：OpenCode 生成的代码质量怎么样？

**A**：通常很好，但有时需要调整。关键是：
- 给出清晰的需求
- 检查生成的代码
- 及时反馈和修改

### Q2：OpenCode 可以处理大项目吗？

**A**：可以，但要分步骤进行。大项目分成多个小任务，逐个完成。

### Q3：如果 OpenCode 理解错了怎么办？

**A**：使用 `/undo` 撤销，然后用更清晰的语言重新描述。

### Q4：OpenCode 生成的代码可以用于生产环境吗？

**A**：可以，但需要：
- 充分测试
- 代码审查
- 性能优化
- 安全检查

### Q5：如何提高 OpenCode 的代码质量？

**A**：
- 给出详细的需求
- 指定技术栈和最佳实践
- 要求添加错误处理
- 要求添加注释和文档
