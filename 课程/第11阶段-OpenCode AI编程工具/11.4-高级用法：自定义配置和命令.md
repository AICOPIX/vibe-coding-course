## ⚙️ 深入配置文件

### 配置文件的完整结构

OpenCode 的配置文件是一个 JSON 文件，可以控制几乎所有行为。

**完整配置示例**：

```json
{
  "$schema": "https://opencode.ai/config.json",
  
  // ========== 基础设置 ==========
  "provider": "anthropic",
  "model": "anthropic/claude-sonnet-4-5",
  "small_model": "anthropic/claude-haiku-4-5",
  
  // ========== 主题和界面 ==========
  "theme": "opencode",
  "autoupdate": true,
  
  // ========== 权限设置 ==========
  "permission": {
    "edit": "auto",      // 自动允许编辑文件
    "bash": "ask",       // 执行命令前询问
    "write": "auto"      // 自动允许写入
  },
  
  // ========== 工具设置 ==========
  "tools": {
    "write": true,       // 允许写入文件
    "edit": true,        // 允许编辑文件
    "bash": true,        // 允许执行 bash 命令
    "read": true         // 允许读取文件
  },
  
  // ========== 自定义命令 ==========
  "command": {
    "test": {
      "template": "运行完整的测试套件，显示覆盖率和失败的测试",
      "description": "运行测试",
      "agent": "build",
      "model": "anthropic/claude-haiku-4-5"
    },
    "component": {
      "template": "创建一个新的 React 组件，名称为 $ARGUMENTS",
      "description": "创建新组件",
      "agent": "build"
    }
  },
  
  // ========== 快捷键设置 ==========
  "keybinds": {
    "submit": "Enter",
    "undo": "Ctrl+Z",
    "redo": "Ctrl+Y"
  },
  
  // ========== 代码格式化 ==========
  "formatter": {
    "prettier": {
      "disabled": false
    }
  },
  
  // ========== MCP 服务器 ==========
  "mcp": {
    "example-server": {
      "type": "stdio",
      "command": "path/to/mcp-server",
      "env": [],
      "args": []
    }
  }
}
```

## 🎮 权限设置详解

### 什么是权限？

权限控制 OpenCode 在执行某些操作时是否需要你的确认。

### 权限选项

| 选项 | 说明 | 例子 |
|------|------|------|
| `auto` | 自动执行，不询问 | OpenCode 直接修改文件 |
| `ask` | 执行前询问 | OpenCode 显示代码，等你确认 |
| `deny` | 禁止执行 | OpenCode 不能执行这个操作 |

### 常见权限设置

#### 设置 1：完全信任（开发者模式）

```json
{
  "permission": {
    "edit": "auto",
    "bash": "auto",
    "write": "auto"
  }
}
```

**适合**：你完全信任 OpenCode，想要最快的速度

#### 设置 2：谨慎模式

```json
{
  "permission": {
    "edit": "ask",
    "bash": "ask",
    "write": "ask"
  }
}
```

**适合**：你想看到每一个修改，确认后再执行

#### 设置 3：平衡模式（推荐）

```json
{
  "permission": {
    "edit": "auto",      // 编辑文件自动执行
    "bash": "ask",       // 执行命令前询问
    "write": "auto"      // 写入文件自动执行
  }
}
```

**适合**：大多数情况，特别是执行命令时要小心

## 🛠️ 创建自定义命令

### 什么是自定义命令？

自定义命令是你为常见任务创建的快捷方式。

**例子**：
- 不用每次都说"运行测试"，直接用 `/test` 命令
- 不用每次都说"创建新组件"，直接用 `/component` 命令

### 创建自定义命令

#### 方法 1：在配置文件中定义

编辑 `~/.config/opencode/opencode.json`：

```json
{
  "command": {
    "test": {
      "template": "运行完整的测试套件，显示覆盖率和失败的测试。如果有失败，建议修复方案。",
      "description": "运行测试",
      "agent": "build",
      "model": "anthropic/claude-haiku-4-5"
    },
    
    "component": {
      "template": "创建一个新的 React 组件，名称为 $ARGUMENTS。包含：\n- TypeScript 类型定义\n- Props 接口\n- 基础结构\n- 导出语句",
      "description": "创建新的 React 组件",
      "agent": "build"
    },
    
    "lint": {
      "template": "运行代码检查工具，找出代码中的问题。显示所有警告和错误。",
      "description": "检查代码质量",
      "agent": "build"
    },
    
    "docs": {
      "template": "为 $ARGUMENTS 文件生成详细的文档注释。包括函数说明、参数说明、返回值说明。",
      "description": "生成文档",
      "agent": "build"
    }
  }
}
```

#### 方法 2：使用 Markdown 文件定义

在 `~/.config/opencode/command/` 目录创建文件。

**例子**：创建 `~/.config/opencode/command/test.md`

```markdown
# test

运行完整的测试套件，显示覆盖率和失败的测试。

如果有失败，建议修复方案。

## 配置

- agent: build
- model: anthropic/claude-haiku-4-5
```

### 使用自定义命令

在 OpenCode 中使用：

```
> /test
```

OpenCode 会执行你定义的命令。

### 命令中的变量

#### `$ARGUMENTS` - 用户输入的参数

```json
{
  "component": {
    "template": "创建一个名为 $ARGUMENTS 的 React 组件"
  }
}
```

**使用**：
```
> /component Button
```

OpenCode 会创建一个名为 "Button" 的组件。

## 🔌 MCP 服务器

### 什么是 MCP？

MCP 是 **Model Context Protocol**（模型上下文协议）。

**简单理解**：
- 像一个"插件系统"
- 让 OpenCode 可以连接外部工具
- 扩展 OpenCode 的功能

### 常见的 MCP 服务器

| 服务器 | 功能 | 用途 |
|--------|------|------|
| `filesystem` | 文件系统访问 | 读写文件 |
| `git` | Git 操作 | 版本控制 |
| `database` | 数据库访问 | 查询数据库 |
| `api` | API 调用 | 调用外部 API |
| `docker` | Docker 操作 | 容器管理 |

### 配置 MCP 服务器

在 `opencode.json` 中添加：

```json
{
  "mcp": {
    "filesystem": {
      "type": "stdio",
      "command": "mcp-server-filesystem",
      "env": [],
      "args": []
    },
    
    "git": {
      "type": "stdio",
      "command": "mcp-server-git",
      "env": [],
      "args": []
    }
  }
}
```

### 使用 MCP 服务器

配置后，OpenCode 会自动使用这些服务器。

**例子**：

```
> 显示最近的 Git 提交记录
```

OpenCode 会使用 Git MCP 服务器来获取提交记录。

## 📝 代码格式化

### 配置代码格式化工具

```json
{
  "formatter": {
    "prettier": {
      "disabled": false
    },
    
    "eslint": {
      "disabled": false
    },
    
    "custom-formatter": {
      "command": ["npx", "prettier", "--write", "$FILE"],
      "environment": {
        "NODE_ENV": "production"
      },
      "extensions": [".js", ".ts", ".jsx", ".tsx"]
    }
  }
}
```

### 说明

- `disabled`: 是否禁用这个格式化工具
- `command`: 执行的命令
- `environment`: 环境变量
- `extensions`: 适用的文件类型

## 🔒 禁用和启用提供商

### 禁用某个提供商

如果你有多个 API 密钥，但只想用某个：

```json
{
  "disabled_providers": ["openai", "groq"]
}
```

这样 OpenCode 就不会使用 OpenAI 和 Groq。

### 只启用某些提供商

```json
{
  "enabled_providers": ["anthropic", "opencode"]
}
```

这样 OpenCode 只会使用 Anthropic 和 OpenCode Zen。

## 📋 项目级配置

### 为特定项目自定义配置

在项目根目录创建 `opencode.json`：

```json
{
  "$schema": "https://opencode.ai/config.json",
  
  // 这个项目使用的模型
  "model": "anthropic/claude-sonnet-4-5",
  
  // 这个项目的权限设置
  "permission": {
    "bash": "ask"  // 执行命令前询问
  },
  
  // 这个项目的自定义命令
  "command": {
    "build": {
      "template": "构建这个项目。运行 npm run build 并检查输出。",
      "description": "构建项目"
    }
  }
}
```

### 配置优先级

项目配置 > 全局配置 > 默认配置

**例子**：
- 全局配置说：使用 Claude Sonnet
- 项目配置说：使用 Claude Haiku
- 结果：这个项目使用 Claude Haiku

## 💡 配置最佳实践

### 1. 从简单开始

不要一开始就配置所有东西。先用默认配置，然后逐步自定义。

### 2. 使用项目配置

为不同的项目创建不同的配置。

### 3. 定期备份

备份你的 `~/.config/opencode/` 目录。

### 4. 记录你的自定义命令

在某个地方记录你创建的自定义命令，方便以后查看。
