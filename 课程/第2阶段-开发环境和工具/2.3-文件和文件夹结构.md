# 2.3 文件和文件夹结构 ☐

> 学习状态：未开始

## 🎯 本节目标

学会认识编程项目的文件夹结构，知道每个文件夹的用途

---

## 📖 为什么要了解文件夹结构？

就像整理房间一样，编程项目也需要把不同的东西放在不同的"抽屉"里：
- 代码放一个地方
- 配置放一个地方
- 素材放一个地方

了解结构后，你就能：
- 快速找到需要的文件
- 知道新文件应该放哪里
- 理解 AI 生成的代码放在哪

---

## 🗂️ 本项目的文件夹结构

```
教学/                          ← 项目根目录（最外层文件夹）
```

## 📁 每个文件夹的作用

### 1. `.kiro/` - Kiro 配置文件夹

```
.kiro/
├── specs/          ← Spec 规格文档（需求、设计、任务）
└── steering/       ← 引导规则（全局规则文件）
```

**用途**：存放 Kiro 的配置和规则
**比喻**：像是"使用说明书"，告诉 Kiro 怎么帮你工作

---

### 2. `.shared/` - 共享资源文件夹

```
.shared/
└── ui-ux-pro-max/  ← UI 设计技能库
    ├── data/       ← 设计数据
    └── scripts/    ← 脚本文件
```

**用途**：存放可复用的资源和工具
**比喻**：像是"工具箱"，放着各种可以重复使用的工具

---

### 3. `.vscode/` - 编辑器配置

**用途**：VS Code / Kiro 编辑器的设置
**比喻**：像是"个人偏好设置"

**注意**：以 `.` 开头的文件夹是"隐藏文件夹"，通常是配置文件

---

### 4. `课程/` - 课程内容

```
课程/
├── 课程目录.md
├── 第1阶段-认识Vibe Coding世界/
│   ├── 1.1-什么是Vibe Coding.md
│   ├── 1.2-AI编程方式全景.md
│   └── ...
├── 第2阶段-开发环境和工具/
└── ...
```

**用途**：存放所有课程的 Markdown 文件
**比喻**：像是"教材书架"，按阶段分类存放

---

### 5. `素材/` - 素材资源

```
素材/
└── UI布局模板/
    ├── README.md
    ├── 01-页面布局.md
    ├── 02-内容区块.md
    └── 03-导航组件.md
```

**用途**：存放可复用的素材、模板
**比喻**：像是"素材库"，需要时可以直接拿来用

---

### 6. `网页/` - 网页文件

```
网页/
├── css/
│   └── style.css           ← 共享样式
├── index.html              ← 首页/目录
├── 1.1-什么是Vibe-Coding.html
└── 6.1-什么是持久化.html
```

**用途**：存放生成的 HTML 网页，方便预览和部署
**比喻**：像是"成品展示区"

---

## 🔍 如何查找文件内容

### 方法1：使用文件树（左侧面板）
点击文件夹展开，找到需要的文件

### 方法2：使用搜索（Ctrl + P）
输入文件名的一部分，快速定位

### 方法3：全局搜索（Ctrl + Shift + F）
搜索文件内容，找到包含特定文字的文件

### 方法4：让 AI 帮你找
直接问 AI："帮我找到关于 xxx 的文件"

## 💡 常见文件夹命名规范

| 文件夹名 | 常见用途 |
|---------|---------|
| `src/` | 源代码（source） |
| `public/` | 公开资源（图片、字体等） |
| `assets/` | 静态资源 |
| `components/` | 组件 |
| `pages/` | 页面 |
| `styles/` 或 `css/` | 样式文件 |
| `scripts/` 或 `js/` | 脚本文件 |
| `images/` 或 `img/` | 图片 |
| `docs/` | 文档 |
| `tests/` | 测试文件 |

---

## 📝 本节小结

1. **项目文件夹像房间的抽屉**：不同东西放不同地方
2. **以 `.` 开头的是隐藏文件夹**：通常是配置文件
3. **了解结构能帮你快速找到文件**
4. **可以用搜索功能快速定位**
