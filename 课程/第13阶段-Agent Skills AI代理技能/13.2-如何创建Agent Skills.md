## 📋 前置要求

在创建 Agent Skills 之前，你需要：

1. **GitHub Copilot** - 已安装并激活
2. **VS Code** - 最新版本（或 VS Code Insiders）
3. **基本的文件操作知识** - 能创建文件夹和文件
4. **中文表达能力** - 能清楚地写出指令

---

## 🏗️ 第一步：创建文件夹结构

### 项目级 Skills（推荐）

如果你想在特定项目中使用 Skill：

```
你的项目/
├── .github/
│   └── skills/
│       └── my-first-skill/
│           ├── SKILL.md
│           └── （其他文件）
```

### 个人 Skills

如果你想在所有项目中使用 Skill：

```
~/.copilot/skills/
└── my-first-skill/
    ├── SKILL.md
    └── （其他文件）
```

或者：

```
~/.claude/skills/
└── my-first-skill/
    ├── SKILL.md
    └── （其他文件）
```

## 📝 第二步：创建 SKILL.md 文件

### 最简单的 SKILL.md

```markdown
---
name: my-first-skill
description: 这是我的第一个 Skill，用来做什么事情
---

# 我的第一个 Skill

## 何时使用此 Skill

当你需要：
- 做什么事情
- 做什么事情

## 具体步骤

1. 第一步
2. 第二步
3. 第三步

## 最佳实践

- 实践 1
- 实践 2
```

### SKILL.md 的三个部分

#### 1️⃣ YAML 前置信息（必需）

```yaml
---
name: skill-name
description: 这个 Skill 的描述
---
```

**要求**：
- `name`：小写，用连字符分隔（例如 `web-testing`）
- `description`：清楚地说明这个 Skill 做什么，何时使用

#### 2️⃣ 标题和简介

```markdown
# Skill 名称

简短的介绍，说明这个 Skill 的目的。
```

#### 3️⃣ 详细内容

```markdown
## 何时使用此 Skill

列出使用场景

## 具体步骤

1. 步骤 1
2. 步骤 2
3. 步骤 3

## 最佳实践

- 实践 1
- 实践 2

## 常见问题

Q: 问题？
A: 答案
```

## 💻 实战例子 1：简单的 Skill

### 场景

你想创建一个 Skill，教 Copilot 如何创建 React 组件。

### 步骤 1：创建文件夹

```bash
mkdir -p .github/skills/react-component-creation
cd .github/skills/react-component-creation
```

### 步骤 2：创建 SKILL.md

```markdown
---
name: react-component-creation
description: 使用 TypeScript 和 Tailwind CSS 创建 React 组件的指南
---

# React 组件创建指南

## 何时使用此 Skill

当你需要：
- 创建新的 React 组件
- 确保组件遵循最佳实践
- 使用 TypeScript 进行类型检查
- 使用 Tailwind CSS 进行样式设计

## 组件结构

每个组件应该包含：

```typescript
import React from 'react';

interface ComponentProps {
  // 定义 Props 类型
}

export const MyComponent: React.FC<ComponentProps> = ({
  // Props
}) => {
  return (
    <div>
      {/* 组件内容 */}
    </div>
  );
};
```

## 创建步骤

1. **定义 Props 接口**
   - 使用 TypeScript 定义所有 Props
   - 添加 JSDoc 注释说明每个 Prop

2. **创建组件函数**
   - 使用函数式组件
   - 使用 React.FC 类型

3. **添加样式**
   - 使用 Tailwind CSS 类名
   - 避免内联样式

4. **导出组件**
   - 使用 named export
   - 添加 display name（用于调试）

## 最佳实践

- ✅ 使用 TypeScript 进行类型检查
- ✅ 为 Props 添加默认值
- ✅ 使用 Tailwind CSS 的响应式类
- ✅ 添加 JSDoc 注释
- ✅ 保持组件小而专注
- ❌ 避免在组件中进行 API 调用
- ❌ 避免使用 any 类型

## 示例

```typescript
interface ButtonProps {
  /** 按钮文本 */
  label: string;
  /** 按钮类型 */
  variant?: 'primary' | 'secondary';
  /** 点击处理函数 */
  onClick?: () => void;
}

export const Button: React.FC<ButtonProps> = ({
  label,
  variant = 'primary',
  onClick,
}) => {
  const baseStyles = 'px-4 py-2 rounded font-semibold';
  const variantStyles = {
    primary: 'bg-blue-500 text-white hover:bg-blue-600',
    secondary: 'bg-gray-200 text-gray-800 hover:bg-gray-300',
  };

  return (
    <button
      className={`${baseStyles} ${variantStyles[variant]}`}
      onClick={onClick}
    >
      {label}
    </button>
  );
};
```
```

### 步骤 3：保存文件

现在你的 Skill 已经创建好了！

```
.github/skills/react-component-creation/
└── SKILL.md
```

## 💻 实战例子 2：包含脚本的 Skill

### 场景

你想创建一个 Skill，教 Copilot 如何运行测试和调试。

### 文件结构

```
.github/skills/testing-guide/
├── SKILL.md
├── test-template.js
└── debug-commands.sh
```

### SKILL.md 内容

```markdown
---
name: testing-guide
description: 使用 Jest 和 Playwright 进行测试的完整指南
---

# 测试指南

## 何时使用此 Skill

当你需要：
- 创建单元测试
- 创建集成测试
- 调试失败的测试

## 单元测试

使用 Jest 进行单元测试。参考 [测试模板](./test-template.js)。

### 运行测试

```bash
npm test
```

### 调试测试

```bash
npm test -- --debug
```

## 集成测试

使用 Playwright 进行集成测试。

### 运行 Playwright 测试

```bash
npx playwright test
```

### 调试 Playwright 测试

```bash
npx playwright test --debug
```

## 常见命令

查看 [调试命令](./debug-commands.sh) 了解更多。

## 最佳实践

- 为每个函数写单元测试
- 为关键用户流程写集成测试
- 使用描述性的测试名称
- 保持测试独立和快速
```

### test-template.js

```javascript
// 测试模板
describe('MyComponent', () => {
  it('应该渲染正确的文本', () => {
    // 测试代码
  });

  it('应该在点击时调用回调函数', () => {
    // 测试代码
  });
});
```

### debug-commands.sh

```bash
#!/bin/bash

# 运行所有测试
npm test

# 运行特定测试文件
npm test -- MyComponent.test.js

# 调试模式
npm test -- --debug

# 查看覆盖率
npm test -- --coverage
```

## 🎯 SKILL.md 的最佳实践

### ✅ 好的 SKILL.md

```markdown
---
name: api-integration
description: 与 REST API 集成的最佳实践和模式
---

# REST API 集成指南

## 何时使用

- 创建 API 调用函数
- 处理错误和重试
- 管理认证令牌

## 步骤

1. 定义 API 端点
2. 创建请求函数
3. 添加错误处理
4. 添加重试逻辑

## 示例

[参考示例文件](./api-example.ts)
```

### ❌ 不好的 SKILL.md

```markdown
---
name: api
description: API 相关
---

# API

做 API 的东西。
```

### 关键要点

- ✅ 清楚的描述（说明何时使用）
- ✅ 具体的步骤
- ✅ 代码示例
- ✅ 参考文件链接
- ❌ 避免过于简洁
- ❌ 避免模糊的描述

## 📂 添加额外文件

### 支持的文件类型

你可以在 Skill 文件夹中添加：

- **Markdown 文件** - 文档和指南
- **代码文件** - 模板和示例（.js, .ts, .py 等）
- **脚本文件** - 自动化脚本（.sh, .bat 等）
- **配置文件** - 配置示例（.json, .yaml 等）

### 在 SKILL.md 中引用文件

```markdown
查看 [测试模板](./test-template.js) 了解如何编写测试。

运行 [设置脚本](./setup.sh) 来初始化项目。

参考 [配置示例](./config.example.json) 了解配置选项。
```

## 🐛 常见错误

### 错误 1：SKILL.md 格式不正确

```markdown
❌ 错误：缺少 YAML 前置信息
name: my-skill
description: 这是我的 Skill

❌ 错误：name 包含大写字母
---
name: MySkill
---

✅ 正确：
---
name: my-skill
description: 这是我的 Skill
---
```

### 错误 2：描述太模糊

```markdown
❌ 错误：
description: 编程相关

✅ 正确：
description: 使用 TypeScript 和 Jest 创建单元测试的指南
```

### 错误 3：文件路径错误

```markdown
❌ 错误：
[模板](test-template.js)  # 文件不存在

✅ 正确：
[模板](./test-template.js)  # 相对路径
```
