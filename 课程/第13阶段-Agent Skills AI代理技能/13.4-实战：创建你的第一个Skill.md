# 13.4 实战：创建你的第一个 Skill

## 🎯 本课目标

通过完整的实战步骤，创建一个真实可用的 Agent Skill。

---

## 📋 实战项目

我们将创建一个 **"Markdown 文档生成"** Skill。

这个 Skill 会教 Copilot 如何：
- 创建结构化的 Markdown 文档
- 遵循文档最佳实践
- 添加目录和导航

---

## 🚀 完整步骤

### 步骤 1：准备工作

#### 1.1 打开你的项目

```bash
# 进入你的项目目录
cd /path/to/your/project

# 用 VS Code 打开
code .
```

#### 1.2 创建文件夹结构

在 VS Code 中：

1. 打开终端（`Ctrl + ~`）
2. 运行命令：

```bash
mkdir -p .github/skills/markdown-documentation
cd .github/skills/markdown-documentation
```

现在你的项目结构应该是：

```
你的项目/
├── .github/
│   └── skills/
│       └── markdown-documentation/
│           └── （我们将在这里创建文件）
```

---

### 步骤 2：创建 SKILL.md 文件

#### 2.1 在 VS Code 中创建文件

1. 在文件浏览器中，右键点击 `markdown-documentation` 文件夹
2. 选择 "New File"
3. 输入文件名：`SKILL.md`

#### 2.2 编写 SKILL.md 内容

复制以下内容到 `SKILL.md`：

```markdown
---
name: markdown-documentation
description: 创建结构化、专业的 Markdown 文档。包含目录、代码示例、最佳实践和导航链接。
---

# Markdown 文档创建指南

## 何时使用此 Skill

当你需要：
- 创建项目文档（README、API 文档等）
- 创建教程或指南
- 创建技术规范文档
- 确保文档结构清晰和专业

## 文档结构

每个 Markdown 文档应该包含以下部分：

### 1. 标题和简介

```markdown
# 文档标题

简短的介绍，说明这个文档的目的。
```

### 2. 目录（可选，用于长文档）

```markdown
## 目录

- [第一部分](#第一部分)
- [第二部分](#第二部分)
- [第三部分](#第三部分)
```

### 3. 主要内容

使用清晰的标题层级：
- `#` - 主标题
- `##` - 章节
- `###` - 小节
- `####` - 子小节

### 4. 代码示例

```markdown
## 代码示例

### JavaScript 示例

\`\`\`javascript
// 这是一个代码示例
function hello() {
  console.log('Hello, World!');
}
\`\`\`

### Python 示例

\`\`\`python
# 这是一个代码示例
def hello():
    print('Hello, World!')
\`\`\`
```

### 5. 表格

```markdown
| 列 1 | 列 2 | 列 3 |
|------|------|------|
| 值 1 | 值 2 | 值 3 |
| 值 4 | 值 5 | 值 6 |
```

### 6. 列表

```markdown
## 有序列表

1. 第一项
2. 第二项
3. 第三项

## 无序列表

- 项目 1
- 项目 2
- 项目 3

## 嵌套列表

- 主项目 1
  - 子项目 1.1
  - 子项目 1.2
- 主项目 2
```

## 创建文档的步骤

1. **确定文档目的**
   - 这个文档是给谁看的？
   - 主要内容是什么？

2. **规划结构**
   - 需要哪些章节？
   - 每个章节的顺序是什么？

3. **编写内容**
   - 使用清晰的语言
   - 添加代码示例
   - 包含实际的使用场景

4. **添加格式**
   - 使用适当的标题级别
   - 添加列表和表格
   - 使用代码块突出代码

5. **检查和优化**
   - 检查拼写和语法
   - 确保链接有效
   - 验证代码示例可以运行

## 最佳实践

- ✅ 使用清晰的标题层级
- ✅ 为代码示例指定语言
- ✅ 添加目录（对于长文档）
- ✅ 使用表格组织信息
- ✅ 包含实际的代码示例
- ✅ 添加链接到相关资源
- ✅ 使用列表组织信息
- ❌ 避免过长的段落
- ❌ 避免没有标题的大块文本
- ❌ 避免不清楚的代码示例

## 常见的文档类型

### README 文档

```markdown
# 项目名称

简短的项目描述。

## 功能

- 功能 1
- 功能 2

## 安装

\`\`\`bash
npm install
\`\`\`

## 使用

\`\`\`javascript
import { MyModule } from 'my-module';
\`\`\`

## 许可证

MIT
```

### API 文档

```markdown
# API 文档

## 端点

### GET /users

获取所有用户。

**参数**：
- `page` (可选) - 页码

**响应**：
\`\`\`json
{
  "users": [
    { "id": 1, "name": "User 1" }
  ]
}
\`\`\`
```

### 教程文档

```markdown
# 如何使用 X

## 前置要求

- 要求 1
- 要求 2

## 步骤 1：安装

\`\`\`bash
npm install x
\`\`\`

## 步骤 2：配置

...

## 步骤 3：使用

...
```

## 参考资源

- [Markdown 官方指南](https://www.markdownguide.org/)
- [GitHub Markdown 指南](https://guides.github.com/features/mastering-markdown/)
```

### 步骤 3：创建参考文件

#### 3.1 创建文档模板

在同一个文件夹中创建 `doc-template.md`：

```markdown
# 文档标题

简短的介绍。

## 目录

- [第一部分](#第一部分)
- [第二部分](#第二部分)

## 第一部分

### 小节 1.1

内容...

### 小节 1.2

内容...

## 第二部分

### 小节 2.1

内容...

## 总结

总结内容...
```

#### 3.2 创建最佳实践指南

创建 `best-practices.md`：

```markdown
# Markdown 文档最佳实践

## 标题

- 使用 `#` 作为主标题
- 使用 `##` 作为章节
- 不要跳过标题级别

## 代码

- 总是指定代码语言
- 使用三个反引号包裹代码
- 确保代码示例可以运行

## 链接

- 使用描述性的链接文本
- 检查链接是否有效

## 列表

- 保持列表项简洁
- 使用一致的格式
```

### 步骤 4：测试你的 Skill

#### 4.1 在 VS Code 中打开 Copilot Chat

按 `Ctrl+Shift+I`（Windows/Linux）或 `Cmd+Shift+I`（macOS）

#### 4.2 测试 Skill

在 Copilot Chat 中输入：

```
> 帮我创建一个项目的 README 文档
> 项目是一个 React 组件库
> 应该包含安装、使用和 API 文档部分
```

#### 4.3 检查结果

Copilot 应该会：
- ✅ 创建结构化的文档
- ✅ 包含清晰的标题
- ✅ 添加代码示例
- ✅ 遵循最佳实践

---

### 步骤 5：优化你的 Skill

#### 5.1 如果 Skill 没有被加载

**问题**：Copilot 没有使用你的 Skill

**解决方案**：
1. 检查文件夹位置是否正确
2. 改进 SKILL.md 的描述
3. 添加更多相关的关键词

#### 5.2 如果结果不满意

**问题**：Copilot 生成的文档不符合预期

**解决方案**：
1. 添加更多具体的步骤
2. 包含更多代码示例
3. 明确列出最佳实践

## 🐛 常见问题

### Q1：Skill 文件夹在哪里？

**A**：
- 项目级：`.github/skills/skill-name/`
- 个人级：`~/.copilot/skills/skill-name/` 或 `~/.claude/skills/skill-name/`

### Q2：如何知道 Skill 是否被加载？

**A**：
- 如果 Copilot 的输出遵循了你的指导，说明被加载了
- 你可以在 Copilot Chat 中看到使用的文件

### Q3：可以有多个 Skills 吗？

**A**：可以！创建多个文件夹，每个文件夹一个 Skill。

### Q4：如何分享我的 Skill？

**A**：
- 上传到 GitHub
- 提交到 [awesome-copilot](https://github.com/github/awesome-copilot)
- 与团队分享
