## 🌟 高级特性

### 1. 跨平台兼容性

Agent Skills 是一个开放标准，可以在多个工具中使用：

#### 支持的工具

| 工具 | 支持情况 | 说明 |
|------|--------|------|
| **VS Code** | ✅ 完全支持 | 最成熟的支持 |
| **GitHub Copilot CLI** | ✅ 完全支持 | 命令行工具 |
| **Copilot Agent** | ✅ 完全支持 | 独立代理 |
| **Visual Studio** | ✅ 支持中 | 正在添加支持 |
| **其他编辑器** | 🔄 计划中 | 未来支持 |

#### 跨平台使用示例

```bash
# 在 VS Code 中使用
code .
# 在 Copilot Chat 中使用你的 Skills

# 在 CLI 中使用
copilot --skill markdown-documentation "创建文档"

# 在 Copilot Agent 中使用
# 自动加载所有可用的 Skills
```

### 2. 组织级 Skills（企业功能）

对于企业用户，可以创建组织级的 Skills：

```
组织级 Skills 位置：
/path/to/organization/.github/skills/

所有团队成员都可以使用这些 Skills
```

**优势**：
- 统一团队的工作流程
- 确保编码标准一致
- 提高团队效率

### 3. 条件加载

你可以创建在特定条件下加载的 Skills：

```markdown
---
name: react-testing
description: 使用 Jest 和 React Testing Library 进行 React 测试
---

# React 测试指南

## 何时使用此 Skill

当你：
- 在 React 项目中工作
- 需要编写单元测试
- 使用 Jest 和 React Testing Library
```

Copilot 会在检测到 React 项目时自动加载这个 Skill。

## 📚 使用社区 Skills

### 步骤 1：浏览可用的 Skills

访问以下仓库查看社区创建的 Skills：

- **anthropics/skills** - Anthropic 官方 Skills
- **github/awesome-copilot** - GitHub 社区集合

### 步骤 2：选择一个 Skill

例如，选择 "web-testing" Skill：

```
https://github.com/anthropics/skills/tree/main/web-testing
```

### 步骤 3：复制到你的项目

```bash
# 进入你的项目
cd /path/to/your/project

# 创建 skills 目录
mkdir -p .github/skills

# 复制 Skill
cp -r /path/to/web-testing .github/skills/
```

### 步骤 4：自定义（可选）

编辑 SKILL.md 以适应你的项目：

```markdown
---
name: web-testing

---

# 网页应用测试指南

## 我们的项目特定配置

我们使用：
- Playwright 进行 E2E 测试
- 自定义的测试基础设施
- 特定的命名规范

...
```

## 🎯 创建高质量 Skills 的技巧

### 1. 明确的使用场景

```markdown
❌ 不好：
description: 编程相关

✅ 好：
description: 使用 TypeScript 和 Jest 为 React 组件编写单元测试。
            当你需要为新组件添加测试覆盖时使用此 Skill。
```

### 2. 包含真实的代码示例

```markdown
✅ 好：
## 示例

```typescript
describe('Button', () => {
  it('应该在点击时调用 onClick', () => {
    const onClick = jest.fn();
    const { getByRole } = render(
      <Button onClick={onClick}>Click me</Button>
    );
    fireEvent.click(getByRole('button'));
    expect(onClick).toHaveBeenCalled();
  });
});
```

❌ 不好：
## 示例
编写一个测试
```

### 3. 提供参考资源

```markdown
✅ 好：
查看 [测试模板](./test-template.ts) 了解标准结构。
参考 [最佳实践](./best-practices.md) 了解推荐做法。

❌ 不好：
查看测试文件了解如何编写测试。
```

### 4. 定期更新

```markdown
# 更新日志

## v2.0 (2025-01-15)
- 添加了 TypeScript 支持
- 更新了 Jest 配置
- 改进了文档

## v1.0 (2024-12-01)
- 初始版本
```

## 📊 Skills 的最佳实践总结

### 设计原则

| 原则 | 说明 | 例子 |
|------|------|------|
| **单一职责** | 每个 Skill 做一件事 | ✅ "React 组件创建" ❌ "所有编程" |
| **清晰描述** | 明确说明何时使用 | ✅ "使用 Jest 编写单元测试" ❌ "测试相关" |
| **包含示例** | 提供真实的代码示例 | ✅ 完整的代码块 ❌ 伪代码 |
| **易于维护** | 结构清晰，易于更新 | ✅ 清晰的章节 ❌ 混乱的内容 |
| **可复用** | 可以在多个项目中使用 | ✅ 通用的最佳实践 ❌ 项目特定的细节 |

### 文件组织

```
skill-name/
├── SKILL.md              ← 核心指导（必需）
├── template.js           ← 代码模板（推荐）
├── examples/             ← 示例文件夹（推荐）
│   ├── example1.js
│   └── example2.js
├── best-practices.md     ← 最佳实践（推荐）
└── README.md             ← 说明文档（可选）
```

## 🔍 常见问题

### Q1：可以有多少个 Skills？

**A**：没有硬性限制，但建议：
- 个人项目：5-10 个
- 团队项目：10-20 个
- 企业组织：20+ 个

### Q2：Skills 会影响性能吗？

**A**：不会。由于三层加载系统，只有相关的 Skills 会被加载。

### Q3：如何更新已有的 Skill？

**A**：
1. 编辑 SKILL.md 文件
2. 添加新的参考文件
3. 更新版本号（可选）
4. 提交更改

### Q4：可以删除 Skill 吗？

**A**：可以。只需删除 Skill 文件夹即可。

### Q5：如何在多个项目中使用同一个 Skill？

**A**：
- 方式 1：在每个项目中复制 Skill
- 方式 2：使用个人级 Skills（`~/.copilot/skills/`）
- 方式 3：使用组织级 Skills（企业功能）
