## 📌 核心概念

**Agent Skills** 是什么？

简单来说，Agent Skills 就像给 AI 助手（Copilot）**教新技能**的方式。

### 生活化比喻

想象你有一个聪明的助手（Copilot），它可以帮你做很多事。但有些特殊的任务，它不太擅长。

**Agent Skills 就像给这个助手一本"操作手册"**：

```
你的需求 → Copilot 查看手册 → 按照手册做事 → 完成任务
```

### 设计师视角

如果你用过 Figma，可以这样理解：

- **Figma 组件库** = 可复用的设计元素
- **Agent Skills** = 可复用的 AI 工作流

就像你可以创建组件库供团队使用，你也可以创建 Skills 供 Copilot 使用。

## 🎯 Agent Skills 能做什么

### 1. 教 Copilot 特定的工作流

```
例子：Web 应用测试 Skill

你告诉 Copilot：
> 帮我创建一个 Playwright 测试

Copilot 加载 "Web 应用测试" Skill，然后：
1. 查看测试模板
2. 按照最佳实践创建测试
3. 使用正确的选择器和断言
```

### 2. 标准化团队的工作方式

```
例子：GitHub Actions 调试 Skill

所有团队成员都使用同一个 Skill，确保：
- 调试流程一致
- 使用相同的工具和命令
- 遵循相同的最佳实践
```

### 3. 提高 Copilot 的准确性

```
例子：项目特定的 Skill

你的项目有特殊的编码规范，创建一个 Skill 告诉 Copilot：
- 如何命名变量
- 如何组织文件
- 如何处理错误
- 如何测试代码
```

## 📊 Agent Skills vs 自定义指令

这两个功能看起来很像，但用途不同：

| 特性 | Agent Skills | 自定义指令 |
|------|-------------|---------|
| **用途** | 教特定的工作流 | 定义编码规范 |
| **内容** | 指令 + 脚本 + 示例 + 资源 | 只有指令 |
| **范围** | 按需加载（任务相关时） | 总是应用 |
| **可移植性** | ✅ 跨 VS Code、CLI、Copilot Agent | ❌ 仅 VS Code 和 GitHub.com |
| **标准** | 开放标准（agentskills.io） | VS Code 专有 |

### 何时使用 Agent Skills

✅ 使用 Agent Skills 当你想要：
- 创建可复用的工作流
- 包含脚本和示例文件
- 在多个工具中使用（VS Code、CLI、Copilot Agent）
- 与团队或社区分享

### 何时使用自定义指令

✅ 使用自定义指令当你想要：
- 定义编码标准（命名规范、代码风格）
- 设置项目信息（技术栈、架构）
- 应用到所有任务（不仅仅是特定任务）

## 🏗️ Agent Skills 的结构

### 文件夹结构

```
.github/skills/
├── webapp-testing/              ← Skill 1
│   ├── SKILL.md                 ← 核心文件（指令）
│   ├── test-template.js         ← 示例文件
│   └── best-practices.md        ← 参考文档
│
└── github-actions-debugging/    ← Skill 2
    ├── SKILL.md
    ├── debug-script.sh
    └── common-issues.md
```

### SKILL.md 文件格式

```markdown
---
name: webapp-testing
description: 使用 Playwright 创建和运行网页应用测试
---

# Web 应用测试指南

## 何时使用此 Skill

当你需要：
- 创建新的 Playwright 测试
- 调试失败的浏览器测试
- 为新项目设置测试基础设施

## 创建测试的步骤

1. 查看 [测试模板](./test-template.js)
2. 确定要测试的用户流程
3. 在 `tests/` 目录创建新测试文件
4. 使用 Playwright 的定位器找到元素
5. 添加断言验证预期行为

## 最佳实践

- 使用 data-testid 属性
- 保持测试独立
- 使用 Page Object Model
```

## 🤖 Copilot 如何使用 Skills

### 三层加载系统

Agent Skills 使用"渐进式加载"，确保高效使用上下文：

#### 第 1 层：发现（总是加载）
```
Copilot 知道有哪些 Skills 可用
（通过读取 SKILL.md 的名称和描述）

这一层很轻量级，不消耗太多上下文
```

#### 第 2 层：加载指令（按需加载）
```
当你的需求匹配某个 Skill 的描述时，
Copilot 加载完整的 SKILL.md 文件

现在 Copilot 可以访问详细的指令
```

#### 第 3 层：访问资源（按需加载）
```
Copilot 只在需要时才访问脚本、示例等文件

这样即使安装了很多 Skills，
也不会浪费上下文空间
```

### 自动激活

你不需要手动选择 Skill，Copilot 会自动决定：

```
你的需求 → Copilot 分析 → 匹配相关 Skill → 自动加载 → 执行任务
```

## 💡 实际例子

### 例子 1：没有 Skill 的情况

```
你：帮我创建一个 Playwright 测试

Copilot：
✓ 创建了一个基础测试
✗ 可能不遵循你的最佳实践
✗ 可能使用了不推荐的选择器
✗ 可能缺少错误处理
```

### 例子 2：有 Skill 的情况

```
你：帮我创建一个 Playwright 测试

Copilot：
1. 检测到这是关于 Playwright 测试
2. 加载 "webapp-testing" Skill
3. 查看测试模板和最佳实践
4. 按照 Skill 中的指导创建测试

✓ 遵循最佳实践
✓ 使用推荐的选择器
✓ 包含错误处理
✓ 符合团队标准
```

## 🚀 为什么要学 Agent Skills

### 1. 提高工作效率

创建一次 Skill，重复使用多次。

### 2. 标准化工作流

确保团队所有人都按照相同的方式工作。

### 3. 改进 AI 输出

给 Copilot 更多信息，让它做得更好。

### 4. 可移植和可分享

你创建的 Skill 可以在多个工具中使用，也可以与社区分享。

### 5. 学习 AI 工作原理

通过创建 Skills，你会更深入地理解 AI 如何工作。

---

## 📝 关键要点

✅ **Agent Skills 是什么**
- 给 Copilot 教新技能的方式
- 包含指令、脚本、示例和资源
- 自动按需加载

✅ **何时使用**
- 需要可复用的工作流
- 需要包含脚本和示例
- 需要跨多个工具使用

✅ **与自定义指令的区别**
- Skills：工作流和流程
- 自定义指令：编码规范和标准
