# 10.6 网格化 UI 设计方案 ☐

## 本节目标
学会用网格坐标精确描述页面布局，让 AI 准确理解你的设计意图

## 核心概念

### 什么是网格化设计？
把页面想象成一个**棋盘**，比如 10×10 的格子。每个格子都有坐标，你可以用坐标来精确描述每个元素的位置和大小。

**生活化比喻**：
- 就像 Excel 表格，A1、B2、C3...
- 就像围棋棋盘，每个位置都有明确的坐标
- 就像城市地图的网格，"在第3街和第5大道的交叉口"

### 为什么要用网格？
| 传统描述 | 网格描述 |
|---------|---------|
| "导航在上面" | "导航：第1行，第1-10列" |
| "侧边栏在右边，窄一点" | "侧边栏：第2-8行，第8-10列" |
| "内容区大一些" | "内容区：第2-8行，第1-7列" |

**网格描述的优势**：精确、无歧义、易调整

## 内容大纲

### 1. 网格系统基础
- 选择网格大小（8×8、10×10、12×12）
- 行和列的概念
- 坐标表示法

### 2. 用颜色标记区域
- 每个颜色代表一个功能区
- 颜色 + 坐标 = 精确布局
- 示例：蓝色=导航，白色=内容，灰色=侧边栏

### 3. 常见布局的网格表示

#### 经典三栏布局（10×10 网格）
```
┌─────────────────────────────────────┐
│ 导航栏（蓝色）：第1行，第1-10列      │
├───────────────────────┬─────────────┤
│                       │             │
│ 主内容（白色）         │ 侧边栏      │
│ 第2-9行，第1-7列      │（灰色）     │
│                       │ 第2-9行     │
│                       │ 第8-10列    │
├───────────────────────┴─────────────┤
│ 页脚（深灰）：第10行，第1-10列       │
└─────────────────────────────────────┘
```

#### 卡片网格布局
```
导航：第1行，第1-10列
卡片1：第2-4行，第1-3列
卡片2：第2-4行，第4-6列
卡片3：第2-4行，第7-9列
...以此类推
```

### 4. 如何向 AI 描述网格布局

**模板**：
```
请创建一个网页，使用 10×10 网格布局：

【导航区】蓝色背景
- 位置：第1行，第1-10列
- 内容：左边是 Logo，右边是菜单

【主内容区】白色背景
- 位置：第2-8行，第1-7列
- 内容：文章列表

【侧边栏】浅灰色背景
- 位置：第2-8行，第8-10列
- 内容：推荐文章、标签云

【页脚】深灰色背景
- 位置：第9-10行，第1-10列
- 内容：版权信息、链接
```

### 5. 进阶技巧
- 响应式网格（手机用 4 列，电脑用 12 列）
- 网格间距设置
- 嵌套网格（大格子里套小格子）

## 小贴士

💡 **选择网格大小的建议**：
- 简单页面：8×8 或 10×10
- 复杂页面：12×12
- 数字越大，控制越精细，但描述也越复杂

💡 **颜色标记建议**：
- 用浅色表示内容区
- 用深色表示导航/页脚
- 用中间色表示侧边栏
- 保持颜色数量在 4-5 种以内
