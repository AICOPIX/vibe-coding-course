# 3.5 如何阅读文档、处理常见问题与 UI 沟通

> 学习状态：已整理
> 所属阶段：第3阶段 AI 协作与需求表达
> 内容说明：合并旧 3.7、3.9、3.10。

## 本节目标

把文档阅读、问题排查和界面表达整合成一套沟通能力。

## 课程摘要

很多问题不是代码错，而是文档没读清、边界没说清、界面意图没表达清。

## 核心要点

- 先读目录、约束和接口，再判断怎么改。
- 常见问题解决要先确认事实，再给解决方案。
- 描述 UI 时要同时说明布局、层次、交互和状态。

## 本节练习

选一个页面，用 4 句话分别描述布局、视觉、交互和响应式要求。

## 学完后你应该能做到

- 用自己的话解释本节核心概念
- 把本节内容放回当前项目里判断它的实际作用
- 在后续页面或项目中继续复用这里的方法
