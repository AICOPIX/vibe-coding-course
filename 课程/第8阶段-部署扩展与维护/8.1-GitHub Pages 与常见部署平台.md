# 8.1 GitHub Pages 与常见部署平台

> 学习状态：已整理
> 所属阶段：第8阶段 部署、扩展与维护
> 内容说明：合并旧 9.1 与 9.2。

## 本节目标

理解静态站点发布的最小路径。

## 课程摘要

部署不是神秘步骤，而是把本地可运行的内容稳定地发布到线上。

## 核心要点

- GitHub Pages 适合静态站点，门槛低且和仓库天然集成。
- 不同部署平台的核心差异在于支持的运行方式和自动化程度。
- 发布前先让目录、入口和资源路径稳定，比选平台更重要。

## 本节练习

解释为什么本项目把 网页 作为唯一发布源目录。

## 学完后你应该能做到

- 用自己的话解释本节核心概念
- 把本节内容放回当前项目里判断它的实际作用
- 在后续页面或项目中继续复用这里的方法
