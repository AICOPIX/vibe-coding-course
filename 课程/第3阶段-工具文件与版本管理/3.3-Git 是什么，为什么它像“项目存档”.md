# 3.3 Git 是什么，为什么它像“项目存档”

> 学习状态：已整理
> 所属阶段：第3阶段 开始接触项目文件、终端和版本管理
> 内容说明：由旧 2.4 迁移而来，作为新阶段占位正文。

## 本节目标

理解版本管理的最小闭环。

## 课程摘要

Git 的作用不是增加复杂度，而是为项目提供可追溯、可比较、可回退的历史。对 AI 协作来说，它最重要的价值之一，就是帮你看清“这轮到底改了什么”。

## 核心要点

- 提交的本质是保存一个清晰的阶段状态。
- 对比改动可以帮助你判断 AI 改了什么，而不是盲目接受。
- 一个小而明确的提交，比一个大而混乱的提交更容易维护。

## 本节练习

写出你认同的一条提交信息规范，例如“只做一件事，并在信息中说明结果”。

## 学完后你应该能做到

- 用自己的话解释本节核心概念
- 把本节内容放回当前项目里判断它的实际作用
- 在后续页面或项目中继续复用这里的方法
