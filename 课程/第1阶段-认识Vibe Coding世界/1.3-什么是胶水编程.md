## 📖 什么是胶水编程？

**胶水编程**（Glue Coding）就是把现成的"积木块"组装起来。

```
┌─────────────────────────────────────────────────────────┐
│                    胶水编程的比喻                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────┐    ┌─────┐    ┌─────┐                        │
│   │登录 │    │数据库│    │支付 │    ← 现成的"积木块"   │
│   │组件 │    │服务 │    │接口 │                        │
│   └──┬──┘    └──┬──┘    └──┬──┘                        │
│      │          │          │                            │
│      └────┬─────┴────┬─────┘                            │
│           │          │                                  │
│      ┌────▼──────────▼────┐                            │
│      │    胶水代码        │    ← AI 帮你写的连接代码   │
│      │  （连接各个部分）   │                            │
│      └────────────────────┘                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

---

## 🏠 生活化比喻

### 比喻1：组装家具

```
Vibe Coding：
你："我要一个书架"
AI：[从零开始做一个书架]

胶水编程：
你："把这个宜家的架子和那个抽屉组合起来"
AI：[帮你找到连接方式，把它们固定在一起]
```

### 比喻2：做菜

```
Vibe Coding：
你："做一道红烧肉"
AI：[从买菜到上桌全部搞定]

胶水编程：
你："把这个预制的红烧肉和那个米饭配在一起，再加个汤"
AI：[帮你搭配、摆盘、调整口味]
```

### 比喻3：搭乐高

```
胶水编程就像搭乐高：
- 积木块是现成的（别人做好的组件/服务）
- 你决定怎么组合
- AI 帮你找到正确的连接方式
```

---

## 🔄 和 Vibe Coding 的区别

| 对比项 | Vibe Coding | 胶水编程 |
|--------|-------------|----------|
| **起点** | 从零开始 | 有现成的部件 |
| **你的角色** | 描述想要什么 | 决定怎么组合 |
| **AI 的角色** | 生成完整代码 | 写连接代码 |
| **需要懂的** | 只需描述需求 | 需要知道有哪些"积木" |
| **适合场景** | 新项目、简单功能 | 整合多个服务/组件 |

```
难度对比：

Vibe Coding    ████░░░░░░  简单
胶水编程       ██████░░░░  中等
传统编程       ██████████  困难
```

## 💬 胶水编程的对话示例

### 示例1：连接支付服务

```
你：我有一个商品页面，想接入微信支付，用户点击购买后能付款

AI：好的，我来帮你：
    1. 在商品页面添加购买按钮
    2. 点击后调用微信支付 API
    3. 支付成功后更新订单状态
    [生成连接代码]
```

### 示例2：整合多个组件

```
你：我想把这个日历组件和待办事项列表连起来，
    点击日期就显示那天的待办

AI：好的，我来写连接代码：
    1. 监听日历的日期点击事件
    2. 根据日期筛选待办数据
    3. 更新待办列表显示
    [生成胶水代码]
```

### 示例3：对接 AI 服务

```
你：我有一个输入框，想让用户输入问题后，
    调用 OpenAI 的 API 返回答案

AI：好的，我来连接：
    1. 获取输入框的内容
    2. 发送到 OpenAI API
    3. 把返回的答案显示出来
    [生成调用代码]
```

## 🎯 什么时候会用到胶水编程？

### 场景1：项目变复杂了

```
一开始：Vibe Coding 做了一个简单网站
后来：想加支付功能 → 需要胶水编程连接支付服务
再后来：想加地图功能 → 需要胶水编程连接地图 API
```

### 场景2：使用别人的服务

```
你想做一个天气 App：
- 天气数据不是你自己有的
- 需要调用天气 API 获取
- 胶水编程帮你连接这个 API
```

### 场景3：整合多个现有功能

```
你有：
- 一个用户系统（之前做的）
- 一个商品系统（之前做的）
- 一个支付服务（第三方的）

胶水编程：把它们组合成一个完整的电商网站
```

- 接入第三方服务（支付、地图、AI 等）
- 整合多个现有组件
- 复用之前的代码

那时候自然就会用到胶水编程了。

## 📝 本节小结

1. **胶水编程** = 把现成的"积木块"组装起来
2. **和 Vibe Coding 的区别**：Vibe 从零开始，胶水是组装现有部件
3. **积木块包括**：第三方服务、开源组件、现有代码
4. **什么时候用**：项目变复杂、需要外部服务、整合多个功能
5. **现在不用急**：先学好 Vibe Coding，需要时再学胶水编程
