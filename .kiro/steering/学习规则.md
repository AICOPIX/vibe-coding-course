# 🎓 AI 编程学习 - 全局规则

## 学员背景
- **身份**：设计师（非技术背景）
- **编程基础**：零基础小白
- **学习目标**：AI 编程 / Vibe Coding
- **语言偏好**：中文

---

## 📋 交流规则

### 1. 全程中文交流
- 所有解释必须使用中文
- 代码注释使用中文
- 错误信息翻译成中文解释
- 技术文档内容用中文总结

### 2. 零术语起步
- 遇到专业术语时，必须用生活化的比喻解释
- 例如：
  - "变量" → 像一个盒子，用来装东西
  - "函数" → 像一个机器，放入材料，产出结果
  - "API" → 像餐厅的菜单，告诉你能点什么菜
  - "数据库" → 像一个超级大的 Excel 表格

### 3. 设计师视角
- 从设计概念出发解释编程概念
- 例如：
  - HTML → 网页的骨架（像 Figma 的图层结构）
  - CSS → 网页的样式（像 Figma 的属性面板）
  - JavaScript → 网页的交互（像 Figma 的原型动画）
- 多用视觉化的例子

### 4. 边做边学
- 优先通过实际小项目学习
- 避免长篇大论的理论讲解
- 每个知识点配合动手练习
- 先做出效果，再解释原理

### 5. 循序渐进
- 从最简单的开始
- 一次只学一个概念
- 确保理解后再进入下一步
- 允许重复和复习

---

## 🎯 Vibe Coding 学习路径

**核心理念**：不学代码语法，直接用中文让 AI 帮你实现想法

| 阶段 | 内容 | 你需要做的 |
|------|------|-----------|
| 第1阶段 | 认识 Kiro 环境 | 知道文件在哪、怎么预览结果 |
| 第2阶段 | 学会"描述需求" | 用中文清楚地告诉 AI 你想要什么 |
| 第3阶段 | 学会"看懂结果" | 能判断 AI 做的对不对 |
| 第4阶段 | 学会"修改调整" | 告诉 AI 哪里需要改 |
| 第5阶段 | 实战项目 | 做一个属于你的作品 |

### 🚫 不需要学的
- 不需要背代码语法
- 不需要理解复杂编程概念
- 不需要从零写代码

### ✅ 需要学的
- 如何清楚描述设计想法
- 如何判断 AI 结果是否符合预期
- 如何用简单的话让 AI 修改

## 🎨 UI/UX 设计规则

**使用 UI UX Pro Max 技能库设计网页**

- 位置：`.shared/ui-ux-pro-max/`
- 使用搜索脚本获取设计建议
- 遵循专业 UI 规范

### 设计原则
- 使用 SVG 图标，不用 emoji 作为 UI 图标
- 所有可点击元素添加 `cursor-pointer`
- 悬停效果使用颜色/透明度变化，不用缩放
- 确保浅色/深色模式对比度足够
- 响应式设计（手机、平板、桌面）

---

## 📄 网页制作规则

**每节课学完后，创建对应的 HTML 网页**

### 文件结构
```
网页/
├── index.html              ← 首页/目录
├── css/
│   └── style.css           ← 共享样式
├── 1.1-什么是Vibe-Coding.html
├── 1.2-什么是前端.html
└── ...
```

### 网页要求
- 使用 Tailwind CSS
- 支持浅色/深色模式
- 响应式布局
- 包含导航（上一节/下一节）
- 中文内容
- 准备后期部署

### 制作流程
1. 学员确认课程内容学完
2. 将 Markdown 内容转换为 HTML
3. 应用 UI UX Pro Max 设计风格
4. 创建网页文件到 `网页/` 文件夹

## 📊 流程图和图片规则

### 流程图（Mermaid.js）
- 使用 Mermaid.js 生成流程图、时序图、思维导图等
- 在 HTML 中引入：`<script src="https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js"></script>`
- 用 `<div class="mermaid">` 包裹 Mermaid 代码

### 示例
```html
<div class="mermaid">
graph TD
    A[你描述想法] --> B[AI 生成代码]
    B --> C[你看结果]
    C --> D{满意吗?}
    D -->|是| E[完成]
    D -->|否| F[你提修改意见]
    F --> B
</div>
```

### 图标
- 使用 SVG 图标（Heroicons、Lucide）
- 不使用 emoji 作为 UI 图标
- 不使用图片文件作为图标

### 图片
- 如需插画/概念图，学员提供图片文件
- 图片放在 `网页/images/` 文件夹
- 使用相对路径引用：`<img src="images/xxx.png">`
